GORM学习

GORM是Go Object Relational Mapping的简写，是Go语言的ORM框架，用来连接数据库的。

官网：https://gorm.io/zh_CN/

下载依赖

go get -u gorm.io/driver/mysql	 # MySQL驱动
go get -u gorm.io/gorm			# GORM框架

使用

// 数据源
dsn := "username:password@tcp(127.0.0.1:3306)/databasename?charset=utf8mb4&parseTime=True&loc=Local"
// 连接数据库
db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{})
if err != nil {
    fmt.Println("无法连接数据库。。。")
    panic(err)
}
  • username：MySQL用户名
  • password：密码
  • databasename：数据库名

使用之前提前在MySQL中创建数据库。

建立模型

type User struct {
    gorm.Model
    Name   string `gorm:"size:30"`
    Age    uint   `gorm:"size:128"`
    Gender string `gorm:"size:4;default:男"`
}

gorm.Model是GORM框架自带的，使用它就可以遵守GORM的规则，也可以不使用。

type Model struct {
    ID        uint `gorm:"primarykey"`
    CreatedAt time.Time
    UpdatedAt time.Time
    DeletedAt DeletedAt `gorm:"index"`
}
`gorm:"size:30"`
`gorm:"size:128"`
`gorm:"size:4;default:男"`

这些是GORM框架的字段标签。上面的意思主要就算设置类型的大小和默认值。详细可以参考官方文档 来做。

创建表

GORM可以使用AutoMigrate方法来自动把Go语言层面的对象（结构体）迁移到MySQL中的数据表，相当于使用了AutoMigrate就可以自动创建表的意思。

// 创建User变量
var user User

// 调用AutoMigrate并传入User变量的地址，以自动迁移到MySQL中
err = db.AutoMigrate(&user)
if err != nil {	// 如果自动迁移失败就直接退出
  fmt.Println("自动迁移错误")
  panic(err)
}

把上面的代码整理运行即可得到数据表。

配置日志

方便查看GORM给我们生成的SQL。

方式一：

var mysqlLogger = logger.Default.LogMode(logger.Info)
db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{
    Logger: mysqlLogger,
})

方式二：

var mysqlLogger = logger.Default.LogMode(logger.Info)
db, err := gorm.Open(mysql.Open(dsn), &gorm.Config{})
db = db.Session(&gorm.Session{Logger: mysqlLogger})

插入数据

插入单个数据

user = User{
  Name: "方大同",
  Age:  40,
}

// 插入数据

if result.Error != nil {
  fmt.Println("查询失败")
} else {
  fmt.Println("插入的新ID: ", user.ID)
}

插入多个数据

users := []*User{
  {Name: "陶喆", Age: 54},
  {Name: "王力宏", Age: 50},
  {Name: "张惠妹", Age: 47},
}
tx := db.Create(users)
if tx.Error != nil {
  fmt.Println("插入失败")
}

指定字段插入数据

users := []*User{
  {Name: "周杰伦", Age: 44},
  {Name: "邓紫棋", Age: 34, Gender: "女"},
}
tx := db.Select("Name", "Age", "Gender").Create(users)
fmt.Println("插入了", tx.RowsAffected, "条数据")

和Select方法起相反功能的方法Omit也可以。只不过Omit是忽略字段插入数据。

批量插入数据并指定批量大小

users := []*User{
    {Name: "李荣浩", Age: 38},
    {Name: "薛之谦", Age: 39},
    {Name: "于文文", Age: 35, Gender: "女"},
}
tx := db.CreateInBatches(users, 5)
fmt.Println("插入了", tx.RowsAffected, "条数据")

要求实际插入的量必须小于等于指定的批量插入大小。

查询数据

单表查询

GORM提供了三个方法来做单表查询，它们之间的区别就在于order by：

  • First：升序查找
  • Take：无序查找
  • Last：降序查找

当查询数据库时它添加了 LIMIT 1 条件，且没有找到记录时，它会返回 ErrRecordNotFound 错误。

First 和 Last 方法将分别查找按主键排序的第一条和最后一条记录。只有将目标结构的指针作为参数传递给这两个方法或使用 db.Model() 指定模型时，这两个方法才会起作用。此外，如果没有为相关模型定义主键，那么模型将按第一个字段排序。

官方示例：

// 获取第一条记录（主键升序）
db.First(&user)
// SELECT * FROM users ORDER BY id LIMIT 1;

// 获取一条记录，没有指定排序字段
db.Take(&user)
// SELECT * FROM users LIMIT 1;

// 获取最后一条记录（主键降序）
db.Last(&user)
// SELECT * FROM users ORDER BY id DESC LIMIT 1;

这里一定要传指针，因为查询只会结果会存放到user变量中，所以只有通过传地址才能写入。

本地示例：

打印方法：

}

避免ErrRecordNotFound错误

如果你想避免ErrRecordNotFound错误，你可以使用Find，比如db.Limit(1).Find(&user)，Find方法可以接受struct和slice的数据。

对单个对象使用Find而不带limit，db.Find(&user)将会查询整个表并且只返回第一个对象，这是性能不高并且不确定的。

var user User
db.Limit(1).Find(&user)
user.show()
---------【User】---------
ID：1
姓名：Cukor
年龄：18
性别：男
-------------------------

主键检索

如果主键是数字，则可以通过使用内联条件使用主键检索对象。在处理字符串时，需要格外小心，以避免 SQL 注入；详情请查看安全 部分。

官方示例：

db.First(&user, 10)
// SELECT * FROM users WHERE id = 10;

db.First(&user, "10")
// SELECT * FROM users WHERE id = 10;

db.Find(&users, []int{1,2,3})
// SELECT * FROM users WHERE id IN (1,2,3);

本地示例：

var users []User
db.First(&users, []int{3, 5, 7})
for _, user := range users {
	user.show()
}

尽管这里参数是一个切片，First方法只会查询到第一个数据，这是因为它底层生成的SQL导致的。

---------【User】---------
ID：3
姓名：方大同
年龄：40
性别：男
-------------------------

如果主键是字符串应该使用下面的方式传递：

db.First(&user, "id = ?", "1b74413f-f3b8-409f-ac47-e8c062e3472a")
// SELECT * FROM users WHERE id = "1b74413f-f3b8-409f-ac47-e8c062e3472a";

当目标对象有一个主键值时，将使用主键构建查询条件，例如：

var user = User{ID: 10}
db.First(&user)
// SELECT * FROM users WHERE id = 10;

var result Userdb.Model(User{ID: 10}).First(&result)
// SELECT * FROM users WHERE id = 10;

注意：如果使用 gorm 的特定字段类型（如 gorm.DeletedAt），它将运行不同的查询来检索对象。

type User struct {
  ID           string `gorm:"primarykey;size:16"`
  Name         string `gorm:"size:24"`
  DeletedAt    gorm.DeletedAt `gorm:"index"`
}

var user = User{ID: 15}
db.First(&user)
/*  
    SELECT * 
    FROM `users` 
    WHERE `users`.`id` = '15' AND `users`.`deleted_at` IS NULL 
    ORDER BY `users`.`id` 
    LIMIT 1
*/

因为GORM的Delete方法实际上是一个伪删除操作。就算在Go程序中执行了db.Delete(&user)，那逻辑上是删除了，但是实际上MySQL这边的数据表并没有真正删除，而是把deleted_at字段做了删除的标记，但是该数据仍然存在。

注意：如果对象的主键ID已经设置，那么不能再在Where方法里写id = ?这类操作。

例如：

var user User{ID: 10}
db.Where("id = ?", 20).First(&user)

这样它会把SQL生成下面的样子：

select * from users id = 10 and id = 20;

很明显这个SQL是查询不出东西的，因为id是主键不可能重复，自然就不可能有既等于10又等于20的情况。

注意 在使用 struct 查询时，GORM 只对非零字段进行查询，这意味着如果字段的值为 0、”‘“、false 或其他零值，则不会用于构建查询条件。例如：

db.Where(&User{Name: "jinzhu", Age: 0}).Find(&users)
// SELECT * FROM users WHERE name = "jinzhu";

性别：女

这个时候可以看到age = 0这个条件没有被生成，所以查询出来的是两条记录，而我们需要的只是age = 0的这条记录。

这种情况下就可以使用String的方法查询：

var users []User
tx := db.Where("name = ? and age = ?", "王菲", 0).Find(&users)

这是因为它们生成的SQL都是一样的：

select * from users where name = '王菲' and age = 0;

当然，如果非要使用Struct的方式写也可以，但是这样写就感觉比较丑了：

db.Where(&User{Name: "王菲"}, "age").Find(&users)

就是把age单独拿出来，然后根据默认值是零值，然后生成对应的SQL，和上面的一样。但是还是不要这样写了，明明都包到User里面了还要单独拿出来就让人觉得很迷。

内联条件

可以把Where方法中的条件内联到First和Find方法。

db.First(&user, "name = ? and age = ?", "cukor", 18)

就是相当于之前写在Where方法里的可以直接写到First和Find方法的参数列表中。

Struct和Map的条件也可以直接放，这里不再赘述。

Not

类似Where方法。它就是Where的取反操作。

示例：

要求使用Not，查询性别女并且年龄在30到35岁之间的记录。

tx := db.Not("gender = ?", "男").Where("age between ? and ?", 30, 35).Find(&users)
/*
select id, name, age, gender
from users
where not gender = '男' and age between 30 and 35;
*/

结果：

---------【User】---------
ID：8
姓名：邓紫棋
年龄：34
性别：女
-------------------------
---------【User】---------
ID：11
姓名：于文文
年龄：35
性别：女
-------------------------
返回 2 条数据

要求使用Not，查询性别女并且年龄不在30到35岁之间的记录。

tx := db.Not("gender = ?", "男").Not("age between ? and ?", 30, 35).Find(&users)
/*
select id, name, age, gender
from users
where not gender = '男' and not age between 30 and 35;
*/

这两个Not方法不能合到一个Not中，否则查询不到我们想要的效果。分出来写，它才会在中间生成and。

Not in：

官方：

// Not In
db.Not(map[string]interface{}{"name": []string{"jinzhu", "jinzhu 2"}}).Find(&users)
// SELECT * FROM users WHERE name NOT IN ("jinzhu", "jinzhu 2");

本地：

tx := db.Not(map[string]interface{}{"name": []string{"王菲", "周杰伦", "于文文"}}).Find(&users)
/*
    SELECT * 
    FROM `users` 
    WHERE `name` NOT IN ('王菲','周杰伦','于文文') AND `users`.`deleted_at` IS NULL
*/

Or

and写在Where里，但是or就得单独拿出来。

官方：

db.Where("role = ?", "admin").Or("role = ?", "super_admin").Find(&users)
// SELECT * FROM users WHERE role = 'admin' OR role = 'super_admin';
// Struct
db.Where("name = 'jinzhu'").Or(User{Name: "jinzhu 2", Age: 18}).Find(&users)
// SELECT * FROM users WHERE name = 'jinzhu' OR (name = 'jinzhu 2' AND age = 18);
// Map
db.Where("name = 'jinzhu'").Or(map[string]interface{}{"name": "jinzhu 2", "age": 18}).Find(&users)
// SELECT * FROM users WHERE name = 'jinzhu' OR (name = 'jinzhu 2' AND age = 18);

本地：

占位符

tx := db.Where("name = ?", "陶喆").Or("name = ?", "David Tao").Find(&users)
/*
SELECT * 
FROM `users` 
WHERE (name = '陶喆' OR name = 'David Tao') AND `users`.`deleted_at` IS NULL
*/

结构体

tx := db.Where("name = ?", "陶喆").Or(User{Name: "David Tao", Age: 54}).Find(&users)
/*
SELECT * 
FROM `users` 
WHERE (name = '陶喆' OR (`users`.`name` = 'David Tao' AND `users`.`age` = 54)) 
	AND `users`.`deleted_at` IS NULL
*/

Map

var users []User

Limit & Offset

Limit 指定获取记录的最大数量 Offset 指定在开始返回记录之前要跳过的记录数量。

官方：

db.Limit(3).Find(&users)
// SELECT * FROM users LIMIT 3;

// 通过 -1 消除 Limit 条件
db.Limit(10).Find(&users1).Limit(-1).Find(&users2)
// SELECT * FROM users LIMIT 10; (users1)
// SELECT * FROM users; (users2)

db.Offset(3).Find(&users)
// SELECT * FROM users OFFSET 3;

db.Limit(10).Offset(5).Find(&users)
// SELECT * FROM users OFFSET 5 LIMIT 10;

// 通过 -1 消除 Offset 条件
db.Offset(10).Find(&users1).Offset(-1).Find(&users2)
// SELECT * FROM users OFFSET 10; (users1)
// SELECT * FROM users; (users2)

注意：Offset方法不能单独使用，要结合Limit方法一起使用。例如官方例子的第9行就会在数据库层面报SQL语法错误。

组查询Group&Having

官方示例：

type result struct {
  Date  time.Time
  Total int

db.Model(&User{}).Select("name, sum(age) as total").Where("name LIKE ?", "group%").Group("name").First(&result)
// SELECT name, sum(age) as total FROM `users` WHERE name LIKE "group%" GROUP BY `name`

db.Model(&User{}).Select("name, sum(age) as total").Group("name").Having("name = ?", "group").Find(&result)
// SELECT name, sum(age) as total FROM `users` GROUP BY `name` HAVING name = "group"

var results []UserResult
