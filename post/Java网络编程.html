Java网络编程

网络编程是指编写运行在多个设备（计算机）的程序，这些设备都通过网络连接起来。它可以实现网络设备的行为和流量控制由独立于网络硬件运行的软件处理。用户可以在控制器上编程、实现对网络的配置、控制和管理。在Java中提供了专门的网络开发程序包——java.net，以便开发者进行网络程序的开发。

在整个通信过程中往往会分为两种端点：服务端和客户端，所以围绕着服务器端和客户端的程序开发就有了两种模式：

  • C/S
  • B/S

C/S（Client/Server，客户端/服务器）和B/S（Browser/Server，浏览器/服务器）是两种常见的软件架构模式。C/S模式是指客户端和服务器之间的结构，通过将任务合理分配到客户端和服务器端，降低了系统的通讯开销，可充分利用两端硬件环境优势。

B/S模式是随着Internet技术的兴起，对C/S结构的一种变化或改进的结构。在这种结构下，用户界面完全通过WWW浏览器实现，一部分事务逻辑在前端实现，但是主要事务逻辑在服务器端实现，节约了开发成本，便于软件维护。

在本章重点讲解C/S结构的程序两种实现：TCP模型和UDP模型。

TCP和UDP

TCP（Transmission Control Protocol，传输控制协议）和UDP（User Datagram Protocol，用户数据报协议）都是互联网传输层协议，用于在网络上传输数据。

TCP是一种面向连接的、可靠的、基于字节流的传输层通信协议。它通过三次握手协议建立连接、四次挥手断开连接。TCP协议的作用是保证数据通信的完整性和可靠性，防止丢包。

UDP是一种无连接的传输层协议，提供面向事务的简单不可靠信息传送服务。UDP协议的主要作用是将网络数据流量压缩成数据包的形式。它传输效率高，但不保证数据的完整性和顺序性。

三次握手与四次挥手

三次握手（Three-way Handshake）是指建立一个TCP连接时，需要客户端和服务器总共发送3个报文。三次握手的目的是连接服务器指定端口，建立TCP连接，并同步连接双方的序列号和确认号，交换TCP窗口大小信息。

第一次握手：客户端将TCP报文标志位SYN置为1，随机产生一个序号值seq=x，并将该数据包发送给服务器端，发送完毕后，客户端进入SYN_SENT状态，等待服务器端确认。

第二次握手：服务器端收到数据包后，如果同意连接，则会向客户端发出确认报文。确认报文中应该ACK=1，SYN=1，确认号是ack=x+1，同时也要为自己初始化一个序列号seq=y，此时，TCP服务器进入了SYN_RCVD状态。

第三次握手：客户端收到确认后，还要向服务器给出确认。确认报文的ACK=1，ack=y+1，自己的序列号seq=x+1，此时，TCP连接建立，客户端和服务器端进入ESTABLISHED状态，完成三次握手，随后客户端与服务器端之间可以开始传输数据了。

四次挥手（Four-way Handshake）是指终止一个TCP连接时，需要客户端和服务器总共发送4个报文。

第一次挥手：客户端发出连接释放报文，并且停止发送数据。释放数据报文首部，FIN=1，其序列号为seq=u（等于前面已经传送过来的数据的最后一个字节的序号加1），此时，客户端进入FIN-WAIT-1状态。

第二次挥手：服务器端接收到连接释放报文后，发出确认报文，ACK=1，ack=u+1，并且带上自己的序列号seq=v，此时，服务端就进入了CLOSE-WAIT状态。

第三次挥手：客户端接收到服务器端的确认请求后，客户端就会进入FIN-WAIT-2状态。等待服务器发送连接释放报文。服务器将最后的数据发送完毕后，就向客户端发送连接释放报文。服务器就进入了LAST-ACK状态。

第四次挥手：客户端收到服务器的连接释放报文后，必须发出确认。ACK=1，ack=w+1。此时客户端就进入了TIME-WAIT状态。但此时TCP连接还未终止。必须要经过2MSL后（最长报文寿命），当客户端撤销相应的TCB后才会进入CLOSED状态。

HTTP

HTTP（超文本传输协议）是一种应用层协议，用于传输超媒体文档，如HTML。它最初是为Web浏览器和Web服务器之间的通信而设计的，但也可用于其他目的。HTTP遵循经典的客户端-服务器模型，客户端打开连接以发出请求，然后等待直到收到响应。HTTP是一种无状态协议，这意味着服务器在两个请求之间不保留任何数据（状态）。

HTTP和TCP之间的联系

HTTP和TCP之间的联系在于HTTP是建立在TCP之上的。HTTP是一种应用层协议，它依赖于传输层协议（如TCP）来传输数据。当您使用Web浏览器访问网页时，浏览器会使用HTTP协议向Web服务器发送请求，然后Web服务器会使用HTTP协议响应该请求。这些HTTP消息通过TCP连接传输，以确保数据可靠地传输。

HTTP和HTTPS之间的联系与区别

HTTP和HTTPS之间的联系在于它们都是用于在Web浏览器和Web服务器之间传输超文本的协议。它们之间的主要区别在于HTTPS使用SSL/TLS协议对传输的数据进行加密，以提供安全性。

当您使用Web浏览器访问一个使用HTTP协议的网站时，您发送到该网站的所有信息（包括您输入的用户名和密码）都是明文传输的，这意味着任何人都可以在网络上传输过程中截获并查看这些信息。而当您访问一个使用HTTPS协议的网站时，您发送到该网站的所有信息都会被加密，这样即使有人截获了这些信息，他们也无法查看其中的内容。

因此，HTTPS比HTTP更安全，它可以保护您在网上传输的敏感信息不被窃取。

Echo程序模型

Echo程序模型是TCP编程模型中的一个经典案例。它的基本思想是客户端通过键盘输入一个信息，把此信息发送给服务器端后，服务器端会将此信息反馈给客户端进行显示。该操作主要通过java.net包的两个类实现：

  • ServerSocket（服务器端）
  • Socket（客户端）

ServerSocket类是Java中用于实现TCP服务器端的类。它提供了一些常用的方法，包括：

  • accept()：监听并接受到此套接字的连接。此方法在连接传入之前一直阻塞。
  • bind(SocketAddress endpoint)：将ServerSocket绑定到特定地址（IP地址和端口号）。
  • close()：关闭此套接字。
  • getInetAddress()：返回此套接字绑定的地址。
  • getLocalPort()：返回此套接字绑定的端口号。
  • setSoTimeout(int timeout)：设置此ServerSocket的超时时间，以毫秒为单位。

Socket类是Java中用于实现TCP客户端的类。它提供了一些常用的方法，包括：

  • connect(SocketAddress endpoint)：将此套接字连接到服务器。
  • close()：关闭此套接字。
  • getInputStream()：返回此套接字的输入流。
  • getOutputStream()：返回此套接字的输出流。
  • setSoTimeout(int timeout)：设置此套接字的超时时间，以毫秒为单位。

Echo通信模型的实现需要通过ServerSocket类在服务器端定义数据监听端口，在没有客户端连接时将一直处于等待连接状态。每一个客户端连接到服务器端后都通过Socket实例描述，通过Socket可以获取客户端输入流与输出流实例，这样就可以利用I/O实现通信。

代码示例：

服务端，EchoServer.java：

package cn.cukor.echo;

import java.io.IOException;
import java.io.PrintStream;
import java.net.ServerSocket;
import java.net.Socket;
import java.util.Scanner;

public class EchoServer {
    public static void main(String[] args) throws IOException {
        // 设置服务器监听端口
        ServerSocket serverSocket = new ServerSocket(9999);
        // 打印提示信息
        System.out.println("等待客户端连接......");
        // 等待客户端连接
        Socket clientSocket = serverSocket.accept();
        System.out.println("客户端已连接......");

        /**
         * 先接收客户端发送来的信息，然后才可以将信息处理后发送回客户端
         */
        // 客户端输入流
        Scanner scanner = new Scanner(clientSocket.getInputStream());
        // 设置分隔符
        scanner.useDelimiter("\\n");
        // 客户端输出流
        PrintStream printStream = new PrintStream(clientSocket.getOutputStream());
        // 循环标记位
        boolean flag = true;
        while (flag) {
            // 有数据接收
            if (scanner.hasNext()) {
                // 接收数据内容
                String value = scanner.next().trim();
                // 结束标记
                if ("bye".equalsIgnoreCase(value)) {
                    // 回应信息
                    printStream.println("再见~~~");
                    // 结束循环
                    flag = false;
                } else {
                    printStream.println("【Server-Echo】：" + value);
                }
            }
        }
        // 关闭输入流
        scanner.close();
        // 关闭输出流
        printStream.close();
        // 关闭客户端
        clientSocket.close();
        // 关闭服务器端
        serverSocket.close();
    }
}

客户端，EchoClient.java：

package cn.cukor.echo;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.io.PrintStream;
import java.net.Socket;
import java.util.Scanner;

public class EchoClient {
    private static final BufferedReader KEYBOARD_INPUT = new BufferedReader(new InputStreamReader(System.in));

    public static String getString(String prompt) throws IOException {
        System.out.println(prompt);
        String str = KEYBOARD_INPUT.readLine();
        return str;
    }

    public static void main(String[] args) throws IOException {
        // 定义服务器端的连接信息
        Socket client = new Socket("localhost", 9999);

        /**
         * 现在的客户端要有输入与输出的操作支持，
         * 所以依然要准备Scanner与PrintWriter
         */
        // 接收服务器端输入内容
        Scanner scanner = new Scanner(client.getInputStream());
        scanner.useDelimiter("\\n");
        // 向服务器端发送内容
        PrintStream printStream = new PrintStream(client.getOutputStream());
        // 循环标记
        boolean flag = true;
        while (flag) {
            String input = getString("请输入要发送的内容：").trim();
            printStream.println(input);
            if (scanner.hasNext()) {
                System.out.println(scanner.next());
            }
            if ("bye".equalsIgnoreCase(input)) {
                flag = false;
            }
        }
        scanner.close();
        printStream.close();
        client.close();
    }
}

回顾一下PrintStream类。

PrintStream类是Java中的一个输出流类，它继承自OutputStream类。PrintStream类提供了一系列的print()和println()方法，用于将各种数据类型（包括字符串、字符、整数、浮点数等）输出到指定的输出流中。

PrintStream类通常用于将数据输出到标准输出（即控制台）或文件中。例如，System.out就是一个PrintStream对象，它用于将数据输出到标准输出。
