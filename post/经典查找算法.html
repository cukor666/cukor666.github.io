经典查找算法

开发环境Linux，vim

编程语言：C语言

顺序查找法

从序列的一端开始遍历到另一端，找到对应元素则返回对应下标（或做其他处理），找不到则返回-1.

==可能查找得到，也可能查找不到==。

时间复杂度是O(n)

ASL = (1+2+3+…+n)/n=(1+n)*n/2/n=(1+n)/2

int querySearch(int arr[],int length,int key) {
    for (int i = 0;i < length; i++) {
        if (key == arr[i]) return i;
    }
    return -1;
}

哨兵法查找

选取头部（尾部）位置做哨兵，从尾部（头部）开始遍历到头部（尾部），找到对应元素则返回对应下标。

原来的内存中有一个单元是不用的，这个单元用来存储哨兵。

==一定会查找得到==。

==和顺序查找是一个意思==。

//0号位置不放原始数据，用于存储哨兵（待查找的数据）
int querySearch(int arr[],int length,int key) {
    arr[0] = key;
    while(arr[0] != arr[--length]);
    return length;
}

时间复杂度是O(n)

二分查找法

前提：==序列有序，并且是顺序存储==。因为链表不能跳着访问。

每一次都是拿要查找的数和子序列的中间位置比较，如果要查找的数比中间位置的小就往左边继续找，如果要查找的数比中间位置的大就往 右边继续找，如果要查找的数和中间位置的数一样大就结束递归返回中间位置的下标，经历了多次这样的寻找后仍然找不到就说明序列中没有这个数直接返回-1.

循环写法：

int binarySearch(int arr[],int left,int right,int key) {
    int mid;
    while (left <= right) {
        mid = left + (right - left)/2;
        if (key < arr[mid]) right = mid - 1;
        else if(key > arr[mid]) left = mid + 1;
        else return mid;
    }
    return -1;
}

递归写法：

int binarySearch(int arr[],int left,int right,int key) {
    if (left > right) return -1;
    int mid = left + (right - left)/2;
	if (key < arr[mid]) binarySearch(arr,left,mid - 1,key);
    else if(key > arr[mid]) binarySearch(arr,mid + 1,right,key);
    else return mid;
}

时间复杂度是O(logn)

1 n/2

2 n/2/2

3 n/2/2/2

…

x = logn

分块查找法

需要一个符号表，存储序列中几个比较特殊的元素，在查找之前先在符号表找一遍，然后再到对应的区域进行查找操作。同样的也是找到就返回对应的下标，否则返回-1.

符号表的结构体描述：

typedef struct Data {
	int key;
	int index;
}Data, ST;
int blocksearch(int arr[], int length, ST* st, int len_st, int key) {
	int left = 0, right = len_st - 1;
	int mid = left + ((right - left) >> 1);
	//二分法查找区间
    while (left < right) {
		if (key < st[mid].key)  	  right = mid; //因为最大值包含在其中，所有不能漏掉right所对应的值
		else if (key > st[mid].key)	  left = mid + 1; //因为left所有对应的比过了，就直接跳开
		else break;
		mid = left + ((right - left) >> 1);
	}
	int begin = st[mid].index;
	int end = mid >= len_st - 1 ? length : st[mid + 1].index;
	//在对应的区间中顺序查找
    while (begin < end) {
		if (key == arr[begin]) return begin;
		begin++;
	}
	return -1;
}

void printElement(int arr[], int length, int index) {
	if (index < 0 || index >= length)
		printf("下标越界\n");
	else
		printf("index = %d\tvalue = %d\n", index, arr[index]);
}

int main() {
	srand((unsigned int)time(NULL));
	int arr[15] = { 5,7,9,1,3 ,24,26,28,22,21,47,57,42,56,37 };
	showArr(arr, 15);
	ST st[3] = {
		{9,0},
		{28,5},
		{57,10}
	};
	int index = blocksearch(arr, 15, st, 3, 50);
	printElement(arr, sizeof(arr) / sizeof(arr[0]), index);
	return 0;
}

O(logk+m)

二叉查找树查找法

解决==链式存储结构==的查找。封装一个有序二叉树，该树的特征是中序遍历是有序的（所以又叫二叉排序树），并且每一颗子树中的父亲都比左子树大并且比右子树小。查找的时候就从根部开始向下查找，如果小于则往左边找，大于则往右边找，等于就直接返回，如果都遍历到底部了仍然没有找到就返回NULL.

类似二分查找。

适合动态增加和删除的查找算法。

时间复杂度：

  • 一般是O(logn)
  • 最坏情况：O(n)

当二叉查找形成一边倒的时候就会出现最坏情况，这个时候的查找变成是链表的顺序查找（没有发挥出二叉树的优势）了，就慢了很多。

二叉查找树的结构体描述：

typedef struct BSTree
{
	int key;
	struct BSTree* lchild;
	struct BSTree* rchild;
}BSTree,Node;

二叉查找树的几个相关函数

Node* createNode(int data);
BSTree* insertNode(BSTree* tree, int data);
Node* deleteNode(BSTree* tree, int key);
Node* searchNode(BSTree* tree, int key);
void show_mid(BSTree* tree);

对应相关函数的实现：

创建二叉查找树：

Node* createNode(int data) {
	Node* newNode = (Node*)malloc(sizeof(Node));
	assert(newNode);
	newNode->key = data;
	newNode->lchild = NULL;
	newNode->rchild = NULL;
	return newNode;
}

向二叉查找树插入数据：

二叉查找树在插入数据的时候，都是把数据插入到树的叶子位置。

BSTree* insertNode(BSTree* tree, int data) {
	if (tree == NULL) return createNode(data);
	Node* pMove = tree;
	Node* pBack = NULL;
	while (pMove) {
		pBack = pMove;
		pMove = data < pMove->key ? pMove->lchild : pMove->rchild;
	}
	if (data < pBack->key)	 	pBack->lchild = createNode(data);
	else 						pBack->rchild = createNode(data);
	return tree;
}

删除时需要的辅助函数：

//找最大节点 最右边
Node* maxKeyNode(BSTree* tree) {
	if (tree == NULL) return NULL;
	while (tree->rchild)
		tree = tree->rchild;
	return tree;
}

在二叉查找树中删除节点：

首先先通过左右指针寻找，key和当前节点的key比较，小了就往左边走，大了就往右边走，等于的时候就停下来，删除它。

删除的时候，使用当前节点的左子树的最大值maxNode覆盖掉当前节点的key，然后再遍历到maxNode的位置将maxNode释放内存。这样做的好处是不需要大幅度改变原来树的结构，理解起来也比较简单。实现的时候采用递归的写法实现，这样比较简单，当然使用循环实现也是可以的，但是使用循环来实现比较麻烦。

Node* deleteNode(BSTree* tree, int key) {
	if (tree == NULL) return NULL;
	if (key < tree->key)		tree->lchild = deleteNode(tree->lchild, key);
	else if (key > tree->key)	tree->rchild = deleteNode(tree->rchild, key);
	else {		//tree对应要删除的节点
		if (tree->lchild != NULL && tree->rchild != NULL) {
			Node* max = maxKeyNode(tree->lchild);
			tree->key = max->key;
			tree->lchild = deleteNode(tree->lchild, max->key);
		}
		else {
			Node* temp = tree;
			tree = tree->lchild ? tree->lchild : tree->rchild;
			free(temp);
		}
	}
	return tree;
}

二叉查找的查找：

Node* searchNode(BSTree* tree, int key) {
	Node* pMove = tree;
	while (pMove && pMove->key != key) {
		if (key < pMove->key)			pMove = pMove->lchild;
		else if (key > pMove->key)		pMove = pMove->rchild;
		else 							break;
	}
	return pMove;
}

树的中序遍历：

void show_mid(BSTree* tree) {
	if (tree == NULL) return;
	show_mid(tree->lchild);
	printf("%d ", tree->key);
	show_mid(tree->rchild);
}

平衡二叉树查找法

解决有序二叉树一边倒（变成顺序查找，失去了二叉树的优势）或高度不平衡的情况导致查找效率低的问题。

==平衡二叉树是树的左右高度差不超过2的二叉查找树==。

因为平衡二叉树由苏联计算机科学家发明，所以又以科学家的名字命名称为AVL树。

关于平衡二叉树，我以前已经单独写过博客了，直接链接到以前的博客即可，链接如下：

哈希表查找

元素的存储位置和其关键字之间建立某种直接关系，那么在进行查找时，就无需做比较或做很少次的比较，按照这种关系直接由关键字找到相应的记录。这就是散列查找法的思想。

H(k) = k%3

散列法的几个术语：

  1. 散列函数和散列地址

    在记录的存储位置address和其关键字key之间建立一个确定的对应关系H，使address=H(key)，称这个对应关系H为散列函数，address为散列地址。

  2. 散列表

    一个有限连续的地址空间，用以存储按散列函数计算得到相应散列地址的数据记录。通常散列表的存储空间是一个一维数组。散列地址是数字下标。

  3. 冲突和同义词

    对不同的关键字可能得到同一个散列地址，即key1 != key2, 而H(key1)=H(key2), 这种现象称为冲突。

存储的数据类型结构体描述：

typedef struct Data {
	int data;
	int index;
}Data;

传统数组版哈希表结构体描述：

typedef struct Hash
{
	Data key;
	int search_count;
}Hash;

哈希函数：以取模为例

int HashFunc(int key, int length) {
	return key % length;
}

创建哈希表：

Hash* createHash(int arr[], int arrLen, int hashLen) {
	Hash* hash = (Hash*)calloc(hashLen, sizeof(Hash));
	assert(hash);
	int address;
	int count;
	for (int i = 0; i < arrLen; i++) {
		address = HashFunc(arr[i], hashLen);
		count = 1;
		while (hash[address].search_count != 0) {
			address = (address + 1) % hashLen;
			count++;
		}
		hash[address].key.data = arr[i];
		hash[address].key.index = i;
		hash[address].search_count = count;
	}
	return hash;
}

打印哈希表：

void printHashTable(Hash* hash, int length) {
	printf("key\tsearch\n");
	for (int i = 0; i < length; i++) {
		printf("%d\t%d\n", hash[i].key.data, hash[i].search_count);
	}
	printf("\n---------------------------\n");
}

哈希查找：

Data hashSearch(Hash* hash, int length, int key) {
	Data ret;
	ret.data = key;
	int address = HashFunc(key, length);
	int flag = address;
	while (hash[address].key.data != key && hash[address].search_count != 0) {
		address = (address + 1) % length;
		if (address == flag)
			break;
	}
	ret.index = hash[address].key.data == key ? address : -1;
	return ret;
}

主函数测试：

int main() {
	int arr[10] = { 15,20,45,6,7,8,16,33,24,67 };
	Hash* hash = createHash(arr, 10, 20);
	printHashTable(hash, 20);
	int key = 28;
	Data result = hashSearch(hash, 20, key);
	printf("找到%d在%d\n", result.data, result.index);
	return 0;
}

红黑树查找法

红黑树是一种特化的AVL树（平衡二叉树），都是在进行插入和删除操作时通过特定操作保持二叉查找树的平衡，从而获得较高的查找性能。

红黑树很复杂，但它的最坏情况运行时间也是非常良好的，并且在实践中是高效的： 它可以在O(log n)时间内做查找，插入和删除。

红黑树和AVL树的比较

查找效率：AVL树查找效率比红黑树高，因为AVL树的左右高度很平衡，而红黑树和AVL树的查找又是基于二叉查找树的，所以AVL树的查找效率比红黑树高。

综合效率：红黑树的效率比AVL树的高，因为AVL树在建立（插入数据）的时候，为了满足树的高度差不超过1，所以频繁的调整导致效率变低，而红黑树调整的没有AVL树频繁，所以综合效率往往是红黑树比AVL高。

红黑树的特征：

红黑树是每个结点都带有颜色属性的二叉查找树，颜色或红色或黑色。 在二叉查找树 强制一般要求以外，对于任何有效的红黑树我们增加了如下的额外要求

  1. 节点非黑即红
  2. 根节点是黑色的
  3. 所有叶子节点是黑色的
  4. 每个红色结点的两个子结点都是黑色。（从每个叶子到根的所有路径上不能有两个连续的红色结点）
  5. 从任一结点到其每个叶子的所有路径都包含相同数目的黑色结点

这些约束强制了红黑树的关键性质: ==从根到叶子的最长的可能路径不多于最短的可能路径的两倍长==。结果是这个树大致上是平衡的。因为操作比如插入、删除和查找某个值的最坏情况时间都要求与树的高度成比例，这个在高度上的理论上限允许红黑树在最坏情况下都是高效的，而不同于普通的二叉查找树。
