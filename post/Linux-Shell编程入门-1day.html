Linux-Shell编程入门-1day

Shell解析器

通过命令可以直接查看Linux的Shell解析器。

[root@WIN-L1ECGP4JN8T ~]# cat /etc/shells
/bin/sh
/bin/bash
/usr/bin/sh
/usr/bin/bash

一般用得比较多的就是bash

bash和sh的关系

sh是bash的软连接（链接），本质就是bash本身。虽然sh比较简短，但一般直接使用bash

Shell脚本格式

第一行指定Shell的解析器，一般使用bash，那就直接这样写。

#!/bin/bash

这个必须是在第一行，而且是必须要写的。从第二行开始才是shell去执行的命令。

第一个Shell程序

#!/bin/bash

echo "HelloWorld"

将文件保存为helloworld.sh，一般Shell脚本都以*.sh结尾，但是也不是必须的，只是一种规范，让人能够直接看得懂。

这个程序执行之后就是在终端返回打印了一个HelloWorld。

编写好了之后可以通过bash解析器来运行，也可以给该文件加上可执行权限运行。

例如：

bash：

bash helloworld.sh

增加可执行权限：

chmod +x helloworld.sh

# 相对路径
./helloworld.sh

# 绝对路径
/root/datas/helloworld.sh	# 根据自己计算机的路径具体情况

系统环境变量

常用系统变量 ： 作用域是整个操作系统或者整个用户，注意：临时的用户环境变量只作用到当前 bash及它的子bash。

$HOME、$PWD、$SHELL、$USER等

打印系统变量：

[root@WIN-L1ECGP4JN8T datas]# echo $HOME

自定义变量

普通变量： 作用域是当前shell（当前的解释器）

在bash中，变量默认类型都是字符串类型，无法直接进行数值运算。

变量的值如果有空格，需要使用双引号或单引号括起来。

[root@WIN-L1ECGP4JN8T datas]# str=HelloWorld
[root@WIN-L1ECGP4JN8T datas]# echo $str

[root@WIN-L1ECGP4JN8T datas]# str = HelloWorld

这样写会把str当成一个可以运行的命令。

[root@WIN-L1ECGP4JN8T datas]# str= HelloWorld
-bash: HelloWorld: command not found

这样写会把HelloWorld当成一个可以运行的命令。

特殊变量：$n

$n （功能描述：n为数字，$0代表该脚本名称，$1-$9代表第一到第九个参数，十以上的参数，十以上的参数需要用大括号包含，如${10}）

parameter.sh David Khalil
[root@WIN-L1ECGP4JN8T datas]# cat parameter.sh

特殊变量：$#

$# （功能描述：获取所有输入参数个数，常用于循环）。

[root@WIN-L1ECGP4JN8T datas]# bash parameter.sh 陶喆 方大同
parameter.sh 陶喆 方大同
2
[root@WIN-L1ECGP4JN8T datas]# cat parameter.sh
#!/bin/bash

特殊变量：$* 和$@

$* （功能描述：这个变量代表命令行中所有的参数，$*把所有的参数看成一个整体）

$@ （功能描述：这个变量也代表命令行中所有的参数，不过$@把每个参数区分对待）

区别：

如果这样传参：陶喆 方大同 王力宏

  • $*这样处理：”陶喆 方大同 王力宏”
  • $@这样处理：”陶喆” “方大同” “王力宏”

视觉效果都一样，但还是有区别的。

[root@WIN-L1ECGP4JN8T datas]# bash parameter.sh 陶喆 方大同 王力宏
parameter.sh 陶喆 方大同
3
陶喆 方大同 王力宏
陶喆 方大同 王力宏
[root@WIN-L1ECGP4JN8T datas]# cat parameter.sh
#!/bin/bash

echo "$0 $1 $2"
echo $#
echo $*
echo $@

特殊变量：$？

$？ （功能描述：最后一次执行的命令的返回状态。

  • 如果这个变量的值为0，证明上一个命令正确执行；
  • 如果这个变量的值为非0（具体是哪个数，由命令自己来决定），则证明上一个命令执行不正确了。）
[root@WIN-L1ECGP4JN8T datas]# bash helloworld.sh
HelloWorld
苏丽珍
[root@WIN-L1ECGP4JN8T datas]# echo $?
0
[root@WIN-L1ECGP4JN8T datas]# bash agagaglabga
bash: agagaglabga: No such file or directory
[root@WIN-L1ECGP4JN8T datas]# echo $?
127

运算符

  • $((变量))或$[变量]：取值
  • expr 表达式运算，后面传的是参数，所以参数之间要有空格。
  • 反引号引起来表示原意

# expr的相乘要使用 \ 来做转义

echo $C

注意：条件非空即为true，[ 666 ]返回true，[] 返回false。

-eq 等于（equal） -ne 不等于（Not equal）

-gt 大于（greater than） -ge 大于等于（greater equal）

流程控制

if判断

if [ 条件判断式 ];then
	程序
fi

或者

if [ 条件判断式 ]
	then
		程序
fi

示例：

[root@WIN-L1ECGP4JN8T datas]# bash test.sh 6
6小于10

[root@WIN-L1ECGP4JN8T datas]# bash test.sh 1

[root@WIN-L1ECGP4JN8T datas]# bash test.sh 11
11大于3
[root@WIN-L1ECGP4JN8T datas]# cat test.sh
#!/bin/bash

if [ $1 -lt 10 ];then
        echo "$1小于10"
fi

if [ $1 -gt 3 ]
        then
                echo "$1大于3"
fi

注意事项：

  1. [ 条件判断式 ]，中括号和条件判断式之间必须有空格
  2. if后要有空格

case语句

case $变量名 in
"值1"）
如果变量的值等于值1，则执行程序1
;;
"值2"）
如果变量的值等于值2，则执行程序2
;;
…省略其他分支…
*）
如果变量的值都不是以上的值，则执行此程序
;;
esac

示例：

[root@WIN-L1ECGP4JN8T datas]# bash test.sh 4
你输入的是4
[root@WIN-L1ECGP4JN8T datas]# bash test.sh 8
你输入的是8
[root@WIN-L1ECGP4JN8T datas]# bash test.sh 10
你输入的是其他值
[root@WIN-L1ECGP4JN8T datas]# cat test.sh
#!/bin/bash

case $1 in
        4)
                echo "你输入的是4"
        ;;

        8)
                echo "你输入的是8"
        ;;

        *)
                echo "你输入的是其他值"
        ;;
esac

注意事项：

  1. case行尾必须为单词“in”，每一个模式匹配必须以右括号“）”结束。
  2. 双分号“;;”表示命令序列结束，相当于java中的break。
  3. 最后的“*）”表示默认模式，相当于java中的default。

funname

求斐波那契数：

[root@WIN-L1ECGP4JN8T datas]# bash fib.sh
fib(0) = 0

fib(3) = 2

[root@WIN-L1ECGP4JN8T datas]# cat fib.sh
#!/bin/bash

function fib() {
        # local关键字用来声明局部变量
        local num=$1
        if (( num == 0 )); then
                echo 0
        elif (( num == 1 )); then
                echo 1
        else

        fi
}

for i in {0..10};do
        echo "fib($i) = $(fib $i)"

或者这样写：

#!/bin/bash

function fib() {
        # local关键字用来声明局部变量
        local num=$1
        if [ $num -eq 0 ]; then
                echo 0
        elif [ $num -eq 1 ]; then
                echo 1
        else
                echo $[ $( fib $[num-1] ) + $( fib $[num-2] ) ]
        fi
}

for i in {0..10};do
        echo "fib($i) = $(fib $i)"
done

取值就使用：$[]，取函数返回值使用$?，而$?表示上次调用的结果，所以可以直接把函数调用写到?的位置，即$(函数名 $参数)

那返回值，也可以使用反引号的方式拿。

上面的代码还可以改写成下面这样：

#!/bin/bash

function fib() {
        # local关键字用来声明局部变量
        local num=$1
        if [ $num -eq 0 ]; then
                echo 0
        elif [ $num -eq 1 ]; then
                echo 1
        else
                echo $[ `fib $[num-1]` + `fib $[num-2]` ]
        fi
}

for i in {0..10};do
        echo "fib($i) = `fib $i`"
done

求阶乘：

[root@WIN-L1ECGP4JN8T datas]# bash jiecheng.sh 5
120
[root@WIN-L1ECGP4JN8T datas]# cat jiecheng.sh
#!/bin/bash


function jiecheng() {
        if [ $# -ne 1 -o $1 -lt 0 ]; then
                echo "参数传递错误"
                return 255
        fi
        local n=$1
        if [ $n -le 1 ]; then
                echo "1"
                return 0
        else
                let pre_n=$1-1
                let res=$n*`jiecheng $pre_n`
                echo $res
                return 0
        fi
}


jiecheng $1

sed

sed是一种流编辑器，它一次处理一行内容。

处理时，把当前处理的行存储在临时缓冲区中，称为“模式 空间”，接着用sed命令处理缓冲区中的内容，处理完成后，把缓冲区的内容送往屏幕。接着处理下一 行，这样不断重复，直到文件末尾。文件内容并没有改变，除非你使用重定向存储输出。

sed [选项参数] ‘command’ filename

  • -e：直接在指令列模式上进行sed的动作编辑。

命令：

  • a：新增，a的后面可以接字串，在下一行出现
  • d：删除
  • s：查找并替换

[root@WIN-L1ECGP4JN8T datas]# cat -n sed.txt
     1  花田里犯了错，说好破晓前忘掉
