Rust结构体

体验结构体

#[derive(Debug)]	// 加上这个才能打印结构体
struct User {
    id: i32,
    name: String,
    age: i32,
}

fn main() {
    let user = User{
        id: 1001,
        name: String::from("Cukor"),
        age: 18,
    };
    println!("user = {:#?}", user);
}
// 程序运行后会有说字段没有被使用过的警告，问题不大。

如果不想引入打印结构体的宏，可以自己封装一个打印的函数来打印结构体的信息：

fn show_user(user: &User) {
    println!("id: {}", user.id);
    println!("name: {}", user.name);
    println!("age: {}", user.age);
}

然后在需要打印的地方调用即可。

结构体打印{:?}和{:#?}的区别

let user = User{
    id: 1001,
    name: String::from("Cukor"),
    age: 18,
};
println!("user = {:?}", user);
println!("user = {:#?}", user);
user = User { id: 1001, name: "Cukor", age: 18 }
user = User {
    id: 1001,
    name: "Cukor",
    age: 18,
}

相当于一个会占一行，一个会换行格式化输出。

构造函数

可以自己封装一个构造函数来初始化结构体：

fn init_user(id: i32, name: String, age: i32) -> User {
    let user = User{
        id: id,
        name: name,
        age: age,
    };
    user
}

let user = init_user(1002, String::from("Cukor"), 19);

当初始化结构体时，字段名和参数名相同时，可以简写：

fn init_user(id: i32, name: String, age: i32) -> User {
    let user = User{
        id,
        name,
        age,
    };
    user
}

使用结构体变量为其他结构体初始化：

let user = init_user(1002, String::from("Cukor"), 19);
show_user(&user);
let user2 = User {
    name: String::from("Alice"),
    ..user  // 最后这里不要写 ,
};
show_user(&user2);

只需要单独为特殊的初始化，其他的直接从另一个结构体变量获取，..变量名且必须放最后，注意在它的后面千万不要加,

注意：这里使用这种简写的方式进行初始化，那么user的所有权就转交给user2了，所以后续不能再使用user，否则就会报错。

元组结构体

就是没有字段名的结构体。

struct Color(i32, i32, i32);
//////////////////////////////
let black = Color(0, 0, 0);
let white = Color(255, 255, 255);
println!("black = {:?}", black);
println!("white = {:?}", white);

方法

虽然声明定义函数和方法的关键字都是fn，但是方法是建立在结构体上的，所以在写方法的时候要指明它是基于哪个结构体的，使用impl关键字。例如之前的打印函数show_user，现在把它改写成User结构体的方法。

#[derive(Debug)]
struct User {
    id: i32,
    name: String,
    age: i32,
}

// 方法都写在这里
impl User {	// 绑定了User结构体
    // 因为是方法了，所以不再需要 _user 来标识，直接叫show
    fn show(&self) {   // self 类似 this，都是指向当前对象
        println!("id: {}", self.id);
        println!("name: {}", self.name);
        println!("age: {}", self.age);
    }
}

fn main() {
    let user = init_user(1003, String::from("Betty"), 25);
    user.show();
}

fn init_user(id: i32, name: String, age: i32) -> User {
    let user = User{
        id,
        name,
        age,
    };
    user
}

关联函数

类似C++、Java的静态函数。因为在Rust中关联函数的第一个参数不是&self，所以它不是方法而是函数，它的特性适合用来做初始化。我们之前也用过这样的函数，例如：String::from。

我们把前面自己封装的构造函数，改写成关联函数：

#[derive(Debug)]
struct User {
    id: i32,
    name: String,
    age: i32,
}

impl User {
    // 关联函数，用来做初始化操作最合适
    fn init(id: i32, name: String, age: i32) -> Self {
        Self {
            id,
            name,
            age
        }
    }

    // 因为是方法了，所以不再需要 _user 来标识，直接叫show
    fn show(&self) {   // self 类似 this，都是指向当前对象
        println!("id: {}", self.id);
        println!("name: {}", self.name);
        println!("age: {}", self.age);
    }
}

fn main() {
    let user = User::init(1003, String::from("Betty"), 25);
    user.show();
}

多个impl

Rust中运行写多个impl块，只要它们都关联同一个结构体，那么多个impl分开写和把所有方法都放在同一个impl的效果是一样的。
