Vue组件的属性和方法

在定义Vue组件时，属性和方法是最重要的两个部分。创建组件时，实现了其内部的data方法，这个方法会返回一个对象，此对象中定义的数据会存储在组件中，并通过响应式的更新原理来影响页面渲染。

方法定义在Vue组件的methods选项中，它与属性一样，可以在组件中访问。

属性基础

在Vue组件中定义的属性数据，我们可以直接使用组件进行调用，这是因为Vue在组织数据时，定义的所有属性都会暴漏在组件中。实际上，这些数据是存储在组件的$data对象中。

也就是说，在data方法中返回的对象里定义的属性，可以通过this.属性名访问，也可以通过this.$data.count访问。但是一般方便就直接使用前者的方式进行访问。

data() {
    return {
        msg: "Hello"
    }
}


console.log(this.msg)
console.log(this.$data.msg)

方法基础

组件的方法被定义在methods选项中。

methods: {
    say() {
        console.log(this.msg)
    }
}

计算属性

前面使用的属性都是存储属性，存储属性的值是直接定义好的，当前属性只是起到存储这些值的作用，在Vue中，与之相对的还有计算属性，计算属性并不是用来存储数据的，而是通过一些计算逻辑来实现实时地维护当前属性的值。就比如说，如果存储属性里存放的是出生日期，那么我们想知道他多少岁，那可以通过计算属性来实现，使用当前时间减去出生日期即可，而不需要再将年龄作为存储属性存放，这样会导致不一致性。

data() {
    return {
        age: 19
    }
},
computed: {
    type() {
        return this.age > 18 ? "成年人" : "未成年人"
    }
}

和普通方法（放在methods中）比，代码运行的效果都是一样的，但是计算属性是基于其所依赖的存储属性的值变化而重新计算的，计算完成后，其结果会被缓存，下次访问计算属性时，只要所依赖的属性没有变，其内的逻辑代码就不会重复执行。而函数不同，每次访问都会重新执行函数内的逻辑代码得到解雇哦。因此，在实际应用中，我们可以根据是否需要缓存这一表中来选择使用计算属性或函数。

属性侦听器

属性侦听是Vue非常强大的功能之一。使用属性侦听器可以方便监听某个属性的变化以完成复杂的业务逻辑。

在定义Vue组件时，可以通过watch选项来定义属性侦听器。

示例：

<div id="App">
   <input type="text" v-model="searchText">
</div>
data() {
    return {
        searchText: ""
    }
},
watch: {
    searchText(newValue, oldValue) {
        console.log("new: " + newValue + ", old: " + oldValue);
        if (newValue.length > 10) {
            alert("文本太长了")
        }
    }
}
