Gin获取参数

获取?后面的键值对的参数。

GET

例如：前端发送一个get请求http://localhost:9000/user?name=zhangsan&age=18，然后我们现在需用通过Gin来获取name和age的值。

r.GET("/user", func(ctx *gin.Context) {
	name := ctx.Query("name") // 字符串直接获取，name="zhangsan"
	s := ctx.Query("age")     // 数字需要转换一下，s="18"
	age, _ := strconv.Atoi(s) // age = 18
	type UserTemp struct {
		Name string
		Age  int
	}
	var user = UserTemp{
		Name: name,
		Age:  age,
	}
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "获取用户姓名和年龄成功",
		"data": user,
	})
})

浏览器：http://localhost:9000/user?name=zhangsan&age=18

结果：

{"code":200,"data":{"Name":"zhangsan","Age":18},"msg":"获取用户姓名和年龄成功"}

上面是通过?的方式获取的，如果使用的是路径的方式获取可以采取下面的方式。

例如：http://localhost:9000/user/zhangsan/18。

然后希望返回的JSON数据和前面的一样。

r.GET("/user/:name/:age", func(ctx *gin.Context) {
	// 注意这里是Param不是Params
	name := ctx.Param("name") // 如果是字符串就直接拿，不是就需要转换
	s := ctx.Param("age")     // s = "18"
	age, _ := strconv.Atoi(s) // age = 18
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "获取用户姓名和年龄成功",
		"data": gin.H{
			"Name": name,
			"Age":  age,
		},
	})
})

http://localhost:9000/user/zhangsan/18

结果：

{"code":200,"data":{"Age":18,"Name":"zhangsan"},"msg":"获取用户姓名和年龄成功"}

顺序这个不要紧，因为返回的时候写法不同，如果返回的写法是一样的，那么JSON就是一样的。

在写路由的时候，路径参数不仅可以写出:name这类的，也可以写*name这种，区别就是*表示的是这个参数是可选的，当然如果出现了*，那它应该是最后一个传参

r.GET("/book/:name/*auther", func(ctx *gin.Context) {
	name := ctx.Param("name")
	auther := ctx.Param("auther")
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "查询书名和作者成功",
		"data": map[string]any{
			"Name":   name,
			"Auther": auther,
		},
	})
})

http://localhost:9000/book/富婆爱上我/

浏览器自动转为中文：

http://localhost:9000/book/%E5%AF%8C%E5%A9%86%E7%88%B1%E4%B8%8A%E6%88%91/

结果：

{"code":200,"data":{"Auther":"/","Name":"富婆爱上我"},"msg":"查询书名和作者成功"}

当然还有一个默认参数的方法：DefaultQuery，和Query一样，只不过多了第二个参数，就是如果查询不到的时候，就把第二个参数的结果返回给调用者。

ctx *gin.Context
name := ctx.Query("name")	// 必须传参
name := ctx.DefaultQuery("name", "张三") // 不必须传参，如果不传参，那name="张三"

同样的POST的请求也有类似的。

POST

单个参数：

r.POST("/user", func(ctx *gin.Context) {
	name := ctx.PostForm("name")
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "提交用户名成功",
		"data": name,
	})
})

多个参数：

r.POST("/book", func(ctx *gin.Context) {
	name := ctx.PostForm("name")
	auther := ctx.PostForm("auther")
	s := ctx.PostForm("price")
	price, _ := strconv.ParseFloat(s, 64)
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "提交书籍成功",
		"data": gin.H{
			"Name":   name,
			"Auther": auther,
			"Price":  price,
		},
	})
})

PUT

和Post一样的方式拿参数。

DELETE

和Get一样的方式拿参数。

活在神话里

Put和Delete只活在神话里。实际用得多的还是Get和Post，难怪它们获取参数都可以和前面两个对应。

参数绑定

POST

但是其实一般使用到POST请求都是提交表单，所以一般从前端发送给后端的都是表单的数据，如果一直使用PostForm来获取数据也没问题，但是这样的话意味着一个POST请求就要写很多个PostForm，再写一个POST的时候，功能还是提交这个表单，然后也要再重复写多个PostForm。所以出现了数据绑定ShouldBind。

r.POST("/bookPost", func(ctx *gin.Context) {
	type Book struct {
		Name   string  `json:"name" form:"name"`
		Auther string  `json:"auther" form:"auther"`
		Price  float64 `json:"price" form:"price"`
	}
	var book Book
	ctx.ShouldBind(&book)
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "添加书籍成功",
		"data": book,
	})
})

当然我这里为了方便写代码，就直接把结构体写到了函数里面，但是实际上结构体的定义应该写到全局的地方。

// 全局
type Book struct {
	Name   string  `json:"name" form:"name"`
	Auther string  `json:"auther" form:"auther"`
	Price  float64 `json:"price" form:"price"`
}

// 路由函数的内部
r.POST("/bookPost", func(ctx *gin.Context) {
	var book Book
	ctx.ShouldBind(&book)
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "添加书籍成功",
		"data": book,
	})
})

不管你怎么写，这个form的结构体标签一定要写。要不然拿不到数据。

而且通过这样写之后，就不需要我们自己手动转换数据，看到Price的值直接是Float64。

GET

同样的GET方法也可以使用绑定参数。

http://localhost:9000/bookGet?name=Go语言圣经&auther=ken&price=99.9

浏览器自动转换

http://localhost:9000/bookGet?name=Go%E8%AF%AD%E8%A8%80%E5%9C%A3%E7%BB%8F&auther=ken&price=99.9

// 全局
type Book struct {
	Name   string  `json:"name" form:"name"`
	Auther string  `json:"auther" form:"auther"`
	Price  float64 `json:"price" form:"price"`
}

r.GET("/bookGet", func(ctx *gin.Context) {
	var book Book
	ctx.ShouldBind(&book)
	ctx.JSON(200, gin.H{
		"code": 200,
		"msg":  "查询数据成功",
		"data": book,
	})
})

结果：

{"code":200,"data":{"name":"Go语言圣经","auther":"ken","price":99.9},"msg":"查询数据成功"}
