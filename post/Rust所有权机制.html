Rust所有权机制

认识所有权

Rust 的核心功能（之一）是 所有权（ownership）。虽然该功能很容易解释，但它对语言的其他部分有着深刻的影响。

Rust 通过所有权系统管理内存，编译器在编译时会根据一系列的规则进行检查。如果违反了任何这些规则，程序都不能编译。在运行时，所有权系统的任何功能都不会减慢程序。

记住Rust所有权规则：

  1. Rust 中的每一个值都有一个 所有者（owner）。
  2. 值在任一时刻有且只有一个所有者。
  3. 当所有者（变量）离开作用域，这个值将被丢弃。

Rust基于所有权的机制就可以自动给内存进行自动释放，无需再由程序员来手动释放。

它其实类似C++的唯一能指针。

所有权转移

所有权机制转移只在堆内存上会出现。普通数据类型并不存在所有权的转移。

let x = 3;
let y = x;
println!("x = {x}")	// ok
println!("y = {y}")	// ok

对于普通数据类型来说，它会拷贝一份新的。而对堆内存的数据来说就发生了所有权的转移：

let str = String::from("Hello");	// 在堆上申请内存
let str2 = str; // 发生了所有权转移
println!("str = {str}"); // 报错，因为str已经把所有权转移给str2

因为str将”Hello”的所有权转移给str2后，它就会自动调用drop函数释放内存。不允许二次释放。

如果实在想要拷贝，那需要调用clone方法。

let str = String::from("Hello");
let str2 = str.clone();
println!("str = {str}"); // ok

所有权与函数

当调用函数的时候，如果参数是指向堆内存的，那也发生了所有权转移。

fn main() {
    let str = String::from("哈哈");
    print_str(str); // 发生了所有权转移
    println!("str = {str}"); // 程序走到这一步会报错
}

fn print_str(str: String) {
    println!("str = {str}");
}

为了不让上面的程序报错，可以在函数调用完后，把所有权返回回给str，这样所有权又回来了，就可以继续使用它。

fn main() {
    let mut str = String::from("哈哈");
    str = print_str(str); // 发生了所有权转移
    println!("str = {str}");	// 此时不会报错
}

fn print_str(str: String) -> String {
    println!("str = {str}");
    str	// 调用完，又将所有权返回出去
}

引用

通过函数返回值返回所有权，这个操作不是很优雅，所以Rust又推出了一种概念——引用。

引用（reference）像一个指针，因为它是一个地址，我们可以由此访问储存于该地址的属于其他变量的数据。

与指针不同，引用确保指向某个特定类型的有效值。

fn main() {
    let str = String::from("哈哈");
    print_str(&str); // 传递的是引用，不发生所有权转移
    println!("str = {str}"); // ok
}

fn print_str(str: &String) {
    println!("str = {str}");
}

引用允许你使用值但不获取其所有权。这样就不会发生所有权的转移，因为根本就没给。就相当于引用就是获取了使用权而已。

解引用使用*.

正如变量默认是不可变的，引用也一样。（默认）不允许修改引用的值。

可变引用

如果想通过借用来修改值，那么首先源头本身一定要是可变的mut，然后传递参数的时候应该传&mut而不是&，函数参数列表也应该接收的是&mut，然后就可以在函数内部进行修改了。

fn main() {
    let mut str = String::from("哈哈");
    print_str(&mut str); // 传递的是引用，不发生所有权转移
    println!("str = {str}"); // ok
}

fn print_str(str: &mut String) {
    str.push_str("呵呵");
    println!("str = {str}");
}

可变引用有一个很大的限制：如果你有一个对该变量的可变引用，你就不能再创建对该变量的引用。这些尝试创建两个 s 的可变引用的代码会失败：

let mut s = String::from("hello");

let r1 = &mut s;
let r2 = &mut s;	// 同一个作用域里不可以有多个可变引用 &mut

println!("{}, {}", r1, r2);	// 报错

也 不能在拥有不可变引用的同时拥有可变引用。

fn main() {
    let mut s = String::from("hello");

    let r1 = &s; // 没问题
    let r2 = &s; // 没问题
    let r3 = &mut s; // 大问题，因为前面有过不可变引用

    println!("{}, {}, and {}", r1, r2, r3);
}

但是，如果前面有不可变引用，但是它们有使用过，然后再来一个可变引用，这样是没有问题的：

fn main() {
    let mut s = String::from("hello");

    let r1 = &s; // 没问题
    let r2 = &s; // 没问题
    println!("{} and {}", r1, r2);	// 这里已经使用过了，所以后续再出现可变引用是可以的
    // 此位置之后 r1 和 r2 不再使用

    let r3 = &mut s; // 没问题
    println!("{}", r3);
}

这语法很是BT。

悬垂指针

悬垂指针就是指向了已经释放的内存的指针。

记住不要这样写：

fn main() {
    let reference_to_nothing = dangle();
}

fn dangle() -> &String { // dangle 返回一个字符串的引用

    let s = String::from("hello"); // s 是一个新字符串

    &s // 返回字符串 s 的引用
} // 这里 s 离开作用域并被丢弃。其内存被释放。
