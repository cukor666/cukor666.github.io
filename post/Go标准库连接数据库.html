Go标准库连接数据库

Go语言连接数据库，可以使用标准库的方式连接，这些Go语言官方给我们提供了操作数据库的接口，但是具体实现需要使用对应的数据库驱动。就相当于Go语言给我们提供了标准，然后具体实现不管。

所以Go语言连接数据，首先需要把对应的数据库的驱动包加载过来，然后才可以连接数据库。

本文以MySQL为例。

下载MySQL驱动包

go get -u github.com/go-sql-driver/mysql

使用MySQL驱动

func Open(driverName, dataSourceName string) (*DB, error)

Open打开一个dirverName指定的数据库，dataSourceName指定数据源，一般至少包括数据库文件名和其它连接必要的信息。

完整示例：

package main

import (
    "database/sql"
    "fmt"
    _ "github.com/go-sql-driver/mysql"
)

func main() {
    // DSN: Data Source Name
    // root：MySQL的用户名
    // cukor：密码，根据自己的情况
    // students：数据库名，在连接之前确保已经有该数据库
    dsn := "root:cukor@tcp(localhost:3306)/students"
    db, err := sql.Open("mysql", dsn)
    if err != nil {
       fmt.Println("数据库连接失败")
       panic(err)
    }
    defer db.Close()

    fmt.Println("数据库连接成功")
}

初始化连接

所以上面的代码会让人感觉好像连接数据库成功了，但其实还不是。

Open函数可能只是验证其参数格式是否正确，实际上并不创建与数据库的连接。如果要检查数据源的名称是否真实有效，应该调用Ping方法。

返回的DB对象可以安全地被多个goroutine并发使用，并且维护其自己的空闲连接池。因此，Open函数应该仅被调用一次，很少需要关闭这个DB对象。

一般我们连接数据库之后，是要对数据库的表进行CRUD的操作，所以写代码不可能都写在main函数中，所以是要抽离出去的，但是我们现在的连接数据库的代码写在了main函数里面，所以当我们需要做查询操作的时候，我们希望写一个函数抽离出去，那么当前的db是局部的，那就无法通信，所以应当设置db为全局变量，然后使用一个初始化函数进行初始化操作，在main函数中调用它即可。

func initDB() (err error) {
    // DSN: Data Source Name
    dsn := "root:cukor@tcp(localhost:3306)/students?charset=utf8mb4&parseTime=True"
    db, err = sql.Open("mysql", dsn)
    if err != nil {
       fmt.Println("数据库连接失败")
       return err
    }
    // 尝试与数据库建立连接（校验dsn是否正确）
    err = db.Ping()
    if err != nil {
       return err
    }
    fmt.Println("数据库连接成功")
    return nil
}

然后在main函数中调用，完整示例如下：

package main

import (
	"database/sql"
	"fmt"
	_ "github.com/go-sql-driver/mysql"
)

var db *sql.DB

func initDB() (err error) {
	// DSN: Data Source Name
	dsn := "root:cukor@tcp(localhost:3306)/students?charset=utf8mb4&parseTime=True"
	db, err = sql.Open("mysql", dsn)
	if err != nil {
		fmt.Println("数据库连接失败")
		return err
	}
	// 尝试与数据库建立连接（校验dsn是否正确）
	err = db.Ping()
	if err != nil {
		return err
	}
	fmt.Println("数据库连接成功")
	return nil
}

func main() {
	err := initDB()
	if err != nil {
		fmt.Println("初始化失败")
		panic(err)
	}
}

其中sql.DB是表示连接的数据库对象（结构体实例），它保存了连接数据库相关的所有信息。它内部维护着一个具有零到多个底层连接的连接池，它可以安全地被多个goroutine同时使用。

CRUD

数据准备

准备好一个数据库：

create database students;
use students;

准备好一张表：

create table students.user
(
    id   int auto_increment
        primary key,
    name varchar(50) not null comment '姓名',
    age  int         not null comment '年龄'
);

在go代码层面造一个user结构体：

type user struct {
    ID   int
    Name string
    Age  int
}

在数据库层面先插入几条数据：

insert into user(name, age) values ('Catherine', 20);
insert into user(name, age) values ('Khalil', 17);
insert into user(name, age) values ('Tom', 27);
insert into user(name, age) values ('Alice', 22);

单行查询

调用一下方法，单行查询，结果存放到*Row中

func (db *DB) QueryRow(query string, args ...interface{}) *Row

示例：

func queryOne() {
    sqlStr := `select id, name, age from user where id = ?`
    var u user
    err := db.QueryRow(sqlStr, 2).Scan(&u.ID, &u.Name, &u.Age)
    if err != nil {
       fmt.Printf("查询失败，err: %v\n", err)
       return
    }
    fmt.Printf("id: %d\tname: %s\tage: %d", u.ID, u.Name, u.Age)
}

在main函数中调用，运行后，结果放到了u中，然后打印出来，就是id为2的数据。

多行查询

多行查询db.Query()执行一次查询，返回多行结果（即Rows），一般用于执行select命令。参数args表示query中的占位参数。

示例：

func queryMore() {
    sqlStr := `select id, name, age from user where age > ?`
    rows, err := db.Query(sqlStr, 20)
    if err != nil {
       fmt.Printf("查询失败，err: %v\n", err)
       return
    }
    // 关闭rows释放持有的数据库连接
    defer rows.Close()

    // 现在查询到的结果都放到了rows中，我们需要遍历拿到
    for rows.Next() {
       var u user
       err := rows.Scan(&u.ID, &u.Name, &u.Age)
       if err != nil {
          fmt.Printf("扫描失败，err: %v\n", err)
          return
       }
       fmt.Printf("id: %d\tname: %s\tage: %d\n", u.ID, u.Name, u.Age)
    }
}
id: 3   name: Tom       age: 27
id: 4   name: Alice     age: 22

插入数据

插入、更新和删除操作都使用Exec方法。

Exec执行一次命令（包括查询、删除、更新、插入等），返回的Result是对已执行的SQL命令的总结。

示例：

func insertRow() {
	sqlStr := `insert into user(name, age) values (?, ?)`
	result, err := db.Exec(sqlStr, "David", 36)
	if err != nil {
		fmt.Printf("插入数据失败, err: %v\n", err)
		return
	}
	// 获取新插入数据的ID
	id, err := result.LastInsertId()
	if err != nil {
		fmt.Printf("获取新插入数据的ID失败，err: %v\n", err)
		return
	}
	fmt.Printf("插入成功，新插入数据的ID为：%d\n", id)
}
插入成功，新插入数据的ID为：5

更新数据

也是通过Exec方法。

示例：

func updateRow() {
    sqlStr := `update user set age = ? where name = ?`
    result, err := db.Exec(sqlStr, 31, "Khalil")
    if err != nil {
       fmt.Printf("更新数据失败, err: %v\n", err)
       return
    }
    // 获取影响的条数
    affected, err := result.RowsAffected()
    if err != nil {
       fmt.Printf("获取影响条数失败，err: %v\n", err)
       return
    }
    fmt.Printf("影响了%d条数据\n", affected)
}

删除数据

同样使用Exec方法。

示例：

func deleteRow() {
	sqlStr := `delete from user where id=?`
	result, err := db.Exec(sqlStr, 3)
	if err != nil {
		fmt.Printf("删除数据失败，err: %v\n", err)
		return
	}
	affected, err := result.RowsAffected()
	if err != nil {
		fmt.Printf("获取影响条数失败，err: %v\n", err)
		return
	}
	fmt.Printf("影响的条数为%d\n", affected)
}
