Java多线程

基本概念

Java多线程编程提供了内置的支持。一条线程指的是进程中一个单一顺序的控制流，一个进程中可以并发多个线程，每条线程并行执行不同的任务。多线程是多任务的一种特别的形式，但多线程使用了更小的资源开销。

在Java中，如果要实现多线程，就必须依靠线程主体类，而java.lang.Thread是Java中负责多线程操作类，只需继承 Thread类 ，就能成为线程主体类，为满足一些特殊要求，也可以通过实现Runnable接口或者Callable接口来完成定义。

进程和线程的联系与区别

进程是程序的一次执行过程，它经历了从代码加载、执行、到执行完毕的一个完整过程；同时也是并发执行的程序在执行过程中分配和管理资源的基本单位，竞争计算机系统资源的基本单位。线程可以理解为进程中的执行的一段程序片段，是进程的一个执行单元，是进程内可调度实体，是比进程更小的独立运行的基本单位，线程也被称为轻量级进程。

进程是程序的一次执行过程，它经历了从代码加载、执行、到执行完毕的一个完整过程。线程是进程中的一个实体，它是进程中的一个执行单元。一个进程可以包含多个线程，这些线程共享进程的资源，如内存和文件句柄。线程共享进程的资源，但每个线程有自己独立的运行栈和程序计数器（PC）。线程之间切换的开销小。

举个例子，你可以把进程想象成一个工厂，而线程就像是工厂里的工人。工厂（进程）提供了工人（线程）所需的资源和设施，而工人（线程）则在工厂（进程）中完成各自的任务。每个工人（线程）都有自己的任务和职责，但他们共享工厂（进程）提供的资源。

多线程的优点

多线程有很多优点，主要包括：

  1. 提高程序的响应性：当程序的一部分被阻塞或需要执行耗时操作时，程序的其他部分仍然可以继续运行，从而提高程序的响应性。

  2. 更好地利用多核处理器：多线程可以充分利用多核处理器的优势，让每个核心都能并行执行不同的线程，从而提高程序的执行效率。

  3. 提高资源利用率：多线程可以让程序在等待某些资源（如I/O操作）时，仍然能够执行其他任务，从而提高资源利用率。

  4. 简化程序设计：多线程可以简化一些复杂问题的解决方案。例如，在网络编程中，使用多线程可以简化服务器端对多个客户端请求的处理。

当然，多线程编程也有一些挑战，如线程同步和安全问题。但是只要正确地使用多线程技术，它能带来很多好处。

实现多线程的准备工作

在Java中使用多线程，你需要了解以下几点：

  1. 线程的创建和启动：Java提供了两种创建线程的方法，一种是继承Thread类，另一种是实现Runnable接口。创建好线程后，可以调用它的start()方法来启动它。

  2. 线程的生命周期：Java线程有新建、就绪、运行、阻塞和死亡五种状态。了解这些状态及其转换条件有助于更好地管理线程。

  3. 线程同步：当多个线程访问共享资源时，需要使用同步机制来避免冲突。Java提供了多种同步机制，如synchronized关键字和Lock接口。

  4. 线程通信：多个线程之间可以通过wait()、notify()和notifyAll()方法进行通信。

  5. 线程池：为了提高程序性能，可以使用线程池来管理线程。Java提供了Executor框架来简化线程池的使用。

以上是在Java中使用多线程时需要准备的一些基础知识。

例子：

下面是一个简单的Java多线程示例，它创建了两个线程，分别打印出0到4和5到9这两组数字。

public class MultiThreadExample {
    public static void main(String[] args) {
        // 创建第一个线程
        Thread thread1 = new Thread(new Runnable() {
            @Override
            public void run() {
                for (int i = 0; i < 5; i++) {
                    System.out.println(i);
                }
            }
        });
        // 启动第一个线程
        thread1.start();

        // 创建第二个线程
        Thread thread2 = new Thread(new Runnable() {
            @Override
            public void run() {
                for (int i = 5; i < 10; i++) {
                    System.out.println(i);
                }
            }
        });
        // 启动第二个线程
        thread2.start();
    }
}

上面的代码中，我们创建了两个线程，分别通过实现Runnable接口来定义它们的任务。然后调用它们的start()方法来启动它们。当这两个线程启动后，它们会并发执行各自的任务。

上面的代码一共有多少个线程？

上面的代码中，我们显式地创建了两个线程，分别是thread1和thread2。但实际上，整个程序运行时一共有三个线程，因为Java程序在运行时会自动创建一个主线程来执行main()方法。

所以，上面的代码实际上一共有三个线程：主线程、thread1和thread2。主线程负责执行main()方法中的代码，而thread1和thread2则分别执行它们各自的任务。

当你运行上面的代码时，程序会创建并启动两个线程，分别打印出0到4和5到9这两组数字。但是，由于线程的执行顺序是不确定的，所以打印出的数字的顺序也是不确定的。

在上面的代码中，我们可以看到，创建完线程之后，是使用该线程对象调用了start()方法，这个是必须的吗?

是的，调用线程对象的start()方法是必须的。当你创建一个线程对象后，它只是一个普通的Java对象，它并不会自动启动。要启动一个线程，你必须调用它的start()方法。

当你调用线程对象的start()方法时，Java虚拟机会为该线程分配必要的系统资源，并调用该线程的run()方法来执行它的任务。注意，你不能直接调用线程对象的run()方法来启动线程，因为这样做只会在当前线程中执行run()方法，而不会启动一个新的线程。

总之，要启动一个线程，你必须调用它的start()方法。

start方法都做了什么事情

当你调用线程对象的start()方法时，会发生以下几件事情：

  1. Java虚拟机会向操作系统请求创建一个新的系统线程。

  2. 操作系统会为这个新线程分配必要的资源，如内存和CPU时间，并将其添加到调度队列中。

  3. Java虚拟机会将这个新线程与你创建的线程对象关联起来。

  4. 当操作系统调度到这个新线程时，Java虚拟机会调用与该线程对象关联的run()方法来执行它的任务。

总之，当你调用线程对象的start()方法时，Java虚拟机会与操作系统协作，为你创建一个新的系统线程，并在适当的时候调用它的run()方法来执行它的任务。

多线程的实现是否一定需要操作系统的支持？

多线程的实现需要操作系统的支持。操作系统负责管理和调度线程，为它们分配CPU时间，以及在它们之间进行切换。

当你在Java程序中创建并启动一个线程时，Java虚拟机会向操作系统请求创建一个新的系统线程。操作系统会为这个新线程分配必要的资源，并将其添加到调度队列中。然后，操作系统会根据调度算法来决定何时执行这个线程。

总之，多线程的实现依赖于操作系统的支持。操作系统负责管理和调度线程，以便它们能够并发执行。

实现多线程的方式

一般一共有3种实现多线程的方式：

  1. 继承Thread类，然后重新run方法
  2. 实现Runnable接口中的run方法
  3. 实现Callable接口的call方法

一般用的比较多的方式是第二种方式。因为采用这种方式可以有效地避免单继承的局限，但是从结构上也需要观察Thread与Runnable的联系。

在IDEA中，你可以通过以下步骤来查看Thread类的源代码：

  1. 在代码编辑器中，将光标移动到Thread类的名称上。
  2. 按住Ctrl键，然后单击Thread类的名称。
  3. IDEA会打开一个新标签页，显示Thread类的源代码。

Thread类是Java语言中表示线程的类。它继承自Object类，并实现了Runnable接口。这意味着你可以通过重写Thread类的run()方法来定义线程的任务。

Runnable接口是一个函数式接口，它只有一个抽象方法：run()。这个方法用于定义线程的任务。当你实现Runnable接口时，必须重写这个方法。

Thread类和Runnable接口之间的关系是：Thread类实现了Runnable接口。这意味着你可以使用两种方式来定义线程的任务：一种是继承Thread类并重写它的run()方法，另一种是实现Runnable接口并重写它的run()方法。

Thread类

下面是一个简单的示例，它演示了如何使用外部类继承Thread类来实现多线程。

首先，我们定义一个外部类MyThread，它继承自Thread类，并重写了它的run()方法来定义线程的任务。

public class MyThread extends Thread {
    private int start;
    private int end;

    public MyThread(int start, int end) {
        this.start = start;
        this.end = end;
    }

    @Override
    public void run() {
        for (int i = start; i < end; i++) {
            System.out.println(i);
        }
    }
}

然后，在主类的主方法中，我们创建MyThread类的对象，并调用它们的start()方法来启动它们。

public class MultiThreadExample {
    public static void main(String[] args) {
        // 创建第一个线程
        MyThread thread1 = new MyThread(0, 5);
        // 启动第一个线程
        thread1.start();

        // 创建第二个线程
        MyThread thread2 = new MyThread(5, 10);
        // 启动第二个线程
        thread2.start();
    }
}

上面的代码中，我们创建了两个MyThread对象，并分别传入不同的参数来定义它们各自的任务。然后我们调用这两个线程对象的start()方法来启动它们。

这种使用外部类继承Thread类的方式与前面提到的使用匿名内部类继承Thread类的方式非常相似。它们都需要重写run()方法来定义线程的任务，但创建线程对象的方式略有不同。

Runnable接口

在Java中，你也可以通过实现Runnable接口来创建一个新的线程。下面是一个简单的示例，它演示了如何使用实现Runnable接口的方式来实现多线程。

首先，我们定义一个类MyRunnable，它实现了Runnable接口，并重写了它的run()方法来定义线程的任务。

public class MyRunnable implements Runnable {
    private int start;
    private int end;

    public MyRunnable(int start, int end) {
        this.start = start;
        this.end = end;
    }

    @Override
    public void run() {
        for (int i = start; i < end; i++) {
            System.out.println(i);
        }
    }
}

然后，在主类的主方法中，我们创建MyRunnable类的对象，并将它们作为参数传递给Thread类的构造函数来创建线程对象。最后，我们调用这些线程对象的start()方法来启动它们。

public class MultiThreadExample {
    public static void main(String[] args) {
        // 创建第一个线程
        Thread thread1 = new Thread(new MyRunnable(0, 5));
        // 启动第一个线程
        thread1.start();

        // 创建第二个线程
        Thread thread2 = new Thread(new MyRunnable(5, 10));
        // 启动第二个线程
        thread2.start();
    }
}

上面的代码中，我们创建了两个MyRunnable对象，并分别传入不同的参数来定义它们各自的任务。然后我们将这两个MyRunnable对象作为参数传递给Thread类的构造函数来创建线程对象。最后，我们调用这两个线程对象的start()方法来启动它们。

这种实现Runnable接口的方式与前面提到的继承Thread类的方式非常相似。它们都需要重写run()方法来定义线程的任务，但创建线程对象的方式略有不同。

Callable接口

在Java中，除了继承Thread类和实现Runnable接口这两种方式外，还有一种实现多线程的方式，那就是使用Callable接口和Future接口。

Callable接口与Runnable接口非常相似，它也只有一个抽象方法：call()。但与Runnable接口不同的是，Callable接口的call()方法可以返回一个值，并且可以抛出异常。

Future接口表示异步计算的结果。当你提交一个Callable任务给ExecutorService时，它会返回一个Future对象。你可以使用这个Future对象来获取Callable任务的执行结果。

下面是一个简单的示例，它演示了如何使用Callable接口和Future接口来实现多线程。

首先，我们定义一个类MyCallable，它实现了Callable接口，并重写了它的call()方法来定义线程的任务。

import java.util.concurrent.Callable;

public class MyCallable implements Callable<Integer> {
    private int start;
    private int end;

    public MyCallable(int start, int end) {
        this.start = start;
        this.end = end;
    }

    @Override
    public Integer call() throws Exception {
        int sum = 0;
        for (int i = start; i < end; i++) {
            sum += i;
        }
        return sum;
    }
}

然后，在主类的主方法中，我们创建MyCallable类的对象，并使用ExecutorService的submit()方法来提交它们。submit()方法会返回一个Future对象，我们可以使用这个Future对象来获取Callable任务的执行结果。

import java.util.concurrent.ExecutionException;
import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.Future;

public class MultiThreadExample {
    public static void main(String[] args) throws ExecutionException, InterruptedException {
        // 创建一个固定大小的线程池
        ExecutorService executorService = Executors.newFixedThreadPool(2);

        // 提交第一个Callable任务
        Future<Integer> future1 = executorService.submit(new MyCallable(0, 5));
        // 获取第一个Callable任务的执行结果
        int result1 = future1.get();
        System.out.println("Result 1: " + result1);

        // 提交第二个Callable任务
        Future<Integer> future2 = executorService.submit(new MyCallable(5, 10));
        // 获取第二个Callable任务的执行结果
        int result2 = future2.get();
        System.out.println("Result 2: " + result2);

        // 关闭线程池
        executorService.shutdown();
    }
}

上面的代码中，我们创建了一个固定大小的线程池，并使用它来管理线程。然后我们创建了两个MyCallable对象，并使用ExecutorService的submit()方法来提交它们。submit()方法会返回一个Future对象，我们可以使用这个Future对象来获取Callable任务的执行结果。

这种使用Callable接口和Future接口的方式与前面提到的继承Thread类和实现Runnable接口的方式非常相似。它们都需要重写某个方法来定义线程的任务，但创建线程对象和获取执行结果的方式略有不同。

多线程的状态

在Java中，线程有以下几种状态：

  1. 新建（New）：当你创建一个线程对象时，它处于新建状态。此时，它还没有开始执行。

  2. 就绪（Runnable）：当你调用线程对象的start()方法时，它会进入就绪状态。此时，它已经准备好执行，只是在等待操作系统分配CPU时间。

  3. 运行（Running）：当操作系统分配给线程CPU时间时，它会进入运行状态。此时，它正在执行它的任务。

  4. 阻塞（Blocked）：当线程等待某些资源（如I/O操作）时，它会进入阻塞状态。此时，它暂停执行，直到资源可用。

  5. 等待（Waiting）：当线程调用wait()、join()或park()方法时，它会进入等待状态。此时，它暂停执行，直到被唤醒。

  6. 计时等待（Timed Waiting）：当线程调用sleep()、wait(long timeout)、join(long millis)或parkNanos()方法时，它会进入计时等待状态。此时，它暂停执行一段时间，直到被唤醒或超时。

  7. 终止（Terminated）：当线程完成它的任务或抛出未捕获的异常时，它会进入终止状态。此时，它已经结束执行。

线程在这些状态之间转换的条件取决于具体情况。例如，当一个处于阻塞状态的线程获得了所需的资源后，它会转换为就绪状态；当一个处于运行状态的线程调用sleep()方法后，它会转换为计时等待状态。

多线程常用的操作方法

在Java中，Thread类提供了许多用于操作线程的方法。下面是一些常用的方法：

  • start()：启动一个新的线程。当你调用线程对象的start()方法时，Java虚拟机会为该线程分配必要的系统资源，并调用该线程的run()方法来执行它的任务。
  • run()：定义线程的任务。当你继承Thread类或实现Runnable接口时，必须重写这个方法来定义线程的任务。
  • sleep(long millis)：使当前线程暂停执行一段时间。当你调用这个方法时，当前线程会进入计时等待状态，直到指定的时间过去后再恢复执行。
  • join()：等待一个线程完成。当你调用一个线程对象的join()方法时，当前线程会进入等待状态，直到该线程对象完成执行后再恢复。
  • interrupt()：中断一个线程。当你调用一个线程对象的interrupt()方法时，它会设置该线程的中断状态。如果该线程正在执行可中断操作（如sleep()或wait()），它会抛出InterruptedException异常。
  • isInterrupted()：检查一个线程是否被中断。当你调用一个线程对象的isInterrupted()方法时，它会返回该线程的中断状态。
  • yield()：暂停当前线程，让其他线程有机会执行。当你调用这个方法时，当前线程会让出CPU时间，让其他处于就绪状态的线程有机会执行。
  • Thread(Runnable target, String name)：实例化线程对象，接收Runnable接口子类对象，同时设置线程名称。
  • setName(String name)：设置线程名称。
  • getName()：获取线程名称。

以上是Java中常用的一些操作线程的方法。

应用示例

下面是一个简单的Java多线程示例。

示例一：

它演示了如何使用两个线程来计算1到100的和。

首先，我们定义一个类MyRunnable，它实现了Runnable接口，并重写了它的run()方法来定义线程的任务。

public class MyRunnable implements Runnable {
    private int start;
    private int end;
    private int result;

    public MyRunnable(int start, int end) {
        this.start = start;
        this.end = end;
    }

    @Override
    public void run() {
        for (int i = start; i <= end; i++) {
            result += i;
        }
    }

    public int getResult() {
        return result;
    }
}

然后，在主类的主方法中，我们创建两个MyRunnable对象，并将它们作为参数传递给Thread类的构造函数来创建线程对象。然后我们调用这些线程对象的start()方法来启动它们，并调用它们的join()方法来等待它们完成。最后，我们获取这两个MyRunnable对象的执行结果，并将它们相加。

public class MultiThreadExample {
    public static void main(String[] args) throws InterruptedException {
        // 创建第一个线程
        MyRunnable runnable1 = new MyRunnable(1, 50);
        Thread thread1 = new Thread(runnable1);
        // 启动第一个线程
        thread1.start();

        // 创建第二个线程
        MyRunnable runnable2 = new MyRunnable(51, 100);
        Thread thread2 = new Thread(runnable2);
        // 启动第二个线程
        thread2.start();

        // 等待两个线程完成
        thread1.join();
        thread2.join();

        // 获取两个线程的执行结果并相加
        int result = runnable1.getResult() + runnable2.getResult();
        System.out.println("Result: " + result);
    }
}

线程礼让

在Java中，sleep()和yield()方法都可以用来暂停当前线程的执行，但它们的作用和使用方式略有不同。

sleep()方法用于使当前线程暂停执行一段时间。当你调用这个方法时，当前线程会进入计时等待状态，直到指定的时间过去后再恢复执行。例如，下面的代码会使当前线程暂停执行1秒钟：

Thread.sleep(1000);

yield()方法用于暂停当前线程，让其他线程有机会执行。当你调用这个方法时，当前线程会让出CPU时间，让其他处于就绪状态的线程有机会执行。但是，需要注意的是，yield()方法只是给其他线程一个执行的机会，并不能保证当前线程一定会暂停执行。例如，下面的代码会使当前线程让出CPU时间：

Thread.yield();

总之，sleep()和yield()方法都可以用来暂停当前线程的执行，但它们的作用和使用方式略有不同。sleep()方法用于使当前线程暂停执行一段时间，而yield()方法用于暂停当前线程，让其他线程有机会执行。

线程优先级

线程优先级是指线程在获取CPU时间时的优先级。在Java中，每个线程都有一个优先级，它决定了线程在获取CPU时间时的优先级。

线程的优先级是一个整数，范围从1到10。数字越大，优先级越高。默认值是5

你可以使用Thread类的setPriority(int newPriority)方法来设置线程的优先级，也可以使用getPriority()方法来获取线程的当前优先级。

需要注意的是，线程的优先级只是一个提示，它并不能保证高优先级的线程一定会比低优先级的线程先执行。操作系统会根据自己的调度算法来决定线程的执行顺序。只是说优先级高的那么它抢占CPU资源的概率就大。

示例一：

下面是一个简单的示例，它演示了如何设置和获取线程的优先级。

public class MultiThreadExample {
    public static void main(String[] args) {
        // 创建一个线程
        Thread thread = new Thread(new Runnable() {
            @Override
            public void run() {
                // 获取当前正在运行的线程
                Thread currentThread = Thread.currentThread();
                // 打印线程的名称和优先级
                System.out.println("Current thread: " + currentThread.getName() + ", priority: " + currentThread.getPriority());
            }
        });
        // 设置线程的优先级
        thread.setPriority(Thread.MAX_PRIORITY);
        // 启动线程
        thread.start();
    }
}

上面的代码中，我们创建了一个线程，并在它的run()方法中使用getPriority()方法来获取它的当前优先级。然后我们调用这个线程对象的setPriority()方法来设置它的优先级为最高，并调用它的start()方法来启动它。

当你运行这段代码时，你会看到程序打印出“Current thread: Thread-0, priority: 10”。这是因为我们在启动线程之前设置了它的优先级为最高（即10）。

线程的同步与死锁

同步锁

线程同步是指在多线程环境下，对共享资源的访问进行协调，以避免出现数据不一致的问题。当多个线程同时访问同一个共享资源时，如果不进行同步，就可能出现数据不一致的问题。

例如，假设有两个线程同时对一个变量进行自增操作。如果不进行同步，那么这两个线程可能会同时读取到变量的初始值，然后都将其加1并写回。这样，变量的最终值只增加了1，而不是期望的2。

为了解决这个问题，我们可以使用线程同步来对共享资源的访问进行协调。在Java中，可以使用synchronized关键字来实现线程同步。当一个线程进入synchronized代码块时，它会获得一个锁。只有当这个线程释放锁后，其他线程才能进入这个synchronized代码块。

synchronized是Java中的一个关键字，它用于实现线程同步。当你在方法或代码块前加上synchronized关键字时，就表示这个方法或代码块是同步的。也就是说，同一时间只能有一个线程进入这个方法或代码块。

当一个线程进入synchronized方法或代码块时，它会获得一个锁。只有当这个线程释放锁后，其他线程才能进入这个synchronized方法或代码块。这样，就可以保证同一时间只有一个线程访问共享资源，从而避免出现数据不一致的问题。

下面是一个简单的示例，它演示了如何使用synchronized关键字来实现线程同步。

public class Counter {
    private int count = 0;

    public synchronized void increment() {
        count++;
    }

    public synchronized int getCount() {
        return count;
    }
}

上面的代码中，我们定义了一个类Counter，它有一个成员变量count和两个方法：increment()和getCount()。这两个方法都被声明为synchronized，表示它们是同步的。

当多个线程同时调用这两个方法时，它们会互相排斥。也就是说，同一时间只能有一个线程进入这两个方法中的一个。这样，就可以保证对count变量的访问是同步的，从而避免出现数据不一致的问题。

下面是使用synchronized关键字的例子：

一个是使用synchronized修饰代码块，另一个是使用synchronized修饰方法。

使用synchronized修饰代码块

public class SynchronizedExample {
    private int count = 0;

    public void incrementCount() {
        synchronized (this) {
            count++;
        }
    }

    public int getCount() {
        return count;
    }
}

在这个例子中，incrementCount方法中的synchronized代码块保证了同一时刻只有一个线程能够访问其中的代码。这样就能保证对count变量的操作是线程安全的。

两种方式都可以实现线程安全，但是使用synchronized修饰代码块可以更精细地控制需要同步的代码范围。

这个例子只有一个主线程，所以不会出现线程安全问题。但是如果有多个线程同时访问SynchronizedExample对象，那么synchronized关键字就能保证对count变量的操作是线程安全的。

死锁

死锁是指两个或多个线程互相等待对方释放锁，导致它们都无法继续执行的情况。死锁通常发生在多个线程同时持有多个锁，并且它们以不同的顺序请求这些锁时。

例如，假设有两个线程A和B，以及两把锁L1和L2。线程A先获取L1锁，然后再请求L2锁；而线程B先获取L2锁，然后再请求L1锁。如果这两个线程同时运行，并且都获取到了它们先请求的那把锁，那么它们就会互相等待对方释放锁，从而导致死锁。

为了避免死锁，可以采取一些措施来防止它的发生。例如，可以规定所有线程必须按照相同的顺序请求锁；或者可以使用超时机制，在等待锁的时间超过一定阈值后放弃等待。

下面是一个简单的 Java 程序，它演示了如何在两个线程之间产生死锁。

public class DeadlockExample {
    public static void main(String[] args) {
        final String resource1 = "resource1";
        final String resource2 = "resource2";
        
        Thread thread1 = new Thread(() -> {
            synchronized (resource1) {
                System.out.println("Thread 1: locked resource 1");
                try { Thread.sleep(100);} catch (Exception e) {}
                synchronized (resource2) {
                    System.out.println("Thread 1: locked resource 2");
                }
            }
        });
        
        Thread thread2 = new Thread(() -> {
            synchronized (resource2) {
                System.out.println("Thread 2: locked resource 2");
                try { Thread.sleep(100);} catch (Exception e) {}
                synchronized (resource1) {
                    System.out.println("Thread 2: locked resource 1");
                }
            }
        });
        
        thread1.start();
        thread2.start();
    }
}

在这个例子中，有两个线程和两个资源。线程 1 首先获得了对 resource1 的锁，然后尝试获得对 resource2 的锁。同时，线程 2 首先获得了对 resource2 的锁，然后尝试获得对 resource1 的锁。这样，两个线程都在等待对方释放锁，从而导致了死锁。

小结

总之，线程同步和死锁是Java多线程编程中非常重要的概念。线程同步用于协调多个线程对共享资源的访问，以避免出现数据不一致的问题；而死锁是指多个线程互相等待对方释放锁，导致它们都无法继续执行的情况。

综合案例：生产者与消费者

在多线程操作中有一个经典的案例程序——生产者与消费者的问题，生产者不断生产，消费者不断取走生产者生产的产品。

数据同步的问题只能够通过同步代码块或同步方法完成。本程序中，生产者和消费者代表着不同的线程对象，所以此时的同步操作应该设置在Message类中，可以将name和money属性设置定义为单独同步方法。

            if (i % 2 == 0) {
                this.message.set("Cukor", 500);
            } else {
                this.message.set("David", 9999);
            }
        }
    }
}

Consumer.java：

package cn.cukor.thread;

public class Consumer implements Runnable {
    private Message message;

    public Consumer(Message message) {
        this.message = message;
    }

    @Override
    public void run() {
        for (int i = 0; i < 50; i++) {
            System.out.println(this.message.get());
        }
    }
}

本程序在Message类中定义了两个同步处理方法，这样使得不同线程在进行公共数据区域操作时都可以保证数据的完整性，解决了数据设置错乱的问题。

Object线程等待与唤醒

重复操作问题的解决需要引入线程的等待与唤醒机制，而这一机制的实现只能依靠Object类完成。在Object类中定义了以下3个方法完成线程的操作。

方法描述
wait()线程等待
wait(long timeout)设置线程等待毫秒数
wait(long timeout, int nanos)设置线程等待毫秒数和纳秒数
notify()唤醒第一个等待线程
notifyAll()唤醒全部等待线程

修改Message类，解决数据的重复设置和重复取出的操作。

Message.java：

package cn.cukor.thread;

/**
 * 生产数据
 * 成员：
 * 姓名
 * 金钱
 */
public class Message {
    private String name;
    private double money;
    private boolean flag = true;
    // flag = true：允许生产
    // flag = false：不允许生产

    public synchronized void set(String name, double money) {
        if (!this.flag) {
            try {
                super.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        this.name = name;
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        this.money = money;
        this.flag = false;
        super.notify();
    }

    public synchronized String get() {
        if (this.flag) {
            try {
                super.wait();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
        try {
            Thread.sleep(100);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        try {
            return this.name + "-->" + this.money;
        } finally {
            this.flag = true;
            super.notify();
        }

    }
}

本程序中追加了一个数据产生于消费操作的控制逻辑成员属性(flag)，通过此属性的值控制实现线程的等待与唤醒处理操作，从而解决了线程重复操作的问题。
