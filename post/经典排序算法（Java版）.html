经典排序算法（Java版）

〇、工具类准备

SortUtil.java

package cn.cukor.sort.utils;

import java.util.List;

public class SortUtil {
    public static <T> void swap(List<T> list, int i, int j) {
        T t = list.get(i);
        list.set(i, list.get(j));
        list.set(j, t);
    }

    public static <T extends Comparable<T>> boolean greater(T a, T b) {
        return a.compareTo(b) > 0;
    }

    public static <T extends Comparable<T>> boolean less(T a, T b) {
        return a.compareTo(b) < 0;
    }

    public static <T extends Comparable<T>> boolean equal(T a, T b) {
        return a.compareTo(b) == 0;
    }
}

MySort.java：

package cn.cukor.sort;

import java.util.List;

public interface MySort<T extends Comparable<T>> {
    void sort(List<T> list);
}

一、冒泡排序

冒泡排序是一种简单的排序算法，它的基本思想是多次遍历待排序的元素，比较相邻的两个元素，如果它们的顺序不正确就交换它们，直到整个序列按照从小到大（或从大到小）的顺序排列。这个过程像气泡一样，较大（或较小）的元素会逐渐浮到序列的末尾（或开头）。

以下是冒泡排序的算法步骤：

  1. 从第一个元素开始，依次比较相邻的两个元素。如果前一个元素比后一个元素大（或小，根据排序要求确定），就交换它们的位置。

  2. 继续比较下一对相邻元素，重复上述操作，直到遍历到序列的倒数第二个元素。此时，最大（或最小）的元素已经被冒泡到了序列的末尾（或开头）。

  3. 重复上述步骤，但不考虑已经排序好的元素，也就是不再比较已经排在正确位置的元素。

  4. 持续执行步骤1和步骤2，每次都将当前未排序序列中的最大（或最小）元素冒泡到合适的位置，直到整个序列都排好序。

  5. 最终，整个序列按照要求的顺序排列完成。

冒泡排序的时间复杂度为$O(n^2)$，其中$n$是待排序元素的个数。它是一种简单但效率较低的排序算法，在处理小型数据集时比较适用，但在大型数据集上性能较差。其他高效的排序算法如快速排序和归并排序通常被更常用于实际应用中。

冒泡排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 冒泡排序
 *  1. 比较相邻的元素。如果第一个比第二个大，就交换他们两个。
 *  2. 如果数组长度为n，则需要比较n-1轮
 *  3. 每轮比较的次数减少1
 * @param <T>
 */
public class BubbleSort<T extends Comparable<T>> implements MySort<T> {

    @Override
    public void sort(List<T> list) {
        int n = list.size();
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - 1 - i; j++) {
                if (SortUtil.greater(list.get(j), list.get(j + 1))){
                    SortUtil.swap(list, j, j + 1);
                }
            }
        }
    }

}

二、选择排序

选择排序（Selection Sort）是一种简单直观的排序算法，其基本思想是在未排序的元素中选择最小（或最大）的元素，然后将其放到已排序部分的末尾。以下是选择排序的算法步骤：

  1. 首先，将整个序列分为两部分：已排序部分和未排序部分。初始时，已排序部分为空，未排序部分包含所有待排序的元素。

  2. 在未排序部分中找到最小（或最大，根据排序要求确定）的元素，并记录其位置。

  3. 将找到的最小（或最大）元素与未排序部分的第一个元素交换位置，将其放到已排序部分的末尾。

  4. 现在，已排序部分增加了一个元素，未排序部分减少了一个元素。重复步骤2和步骤3，直到未排序部分为空。

  5. 最终，整个序列按照要求的顺序排列完成。

选择排序的时间复杂度为$O(n^2)$，其中$n$是待排序元素的个数。与冒泡排序不同，选择排序不涉及大量的元素交换操作，因此在某些情况下性能略高于冒泡排序，但它仍然是一种相对简单的排序算法，不适用于大型数据集。同样，更高效的排序算法如快速排序和归并排序通常更常用于实际应用中。

选择排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 选择排序
 * 1. 如果数组长度是n，则需要比较n-1轮
 * 2. 每一轮比较只需要找到该数组中数值最小的元素的下标minIndex
 * 3. 每一轮比较后，将该轮比较中找到的最小元素与数组中的第i个元素交换
 * 4. 如果i和minIndex相同，则说明该轮比较中，最小元素就是数组中的第i个元素，不需要交换，就算交换了也是和自己交换，不会影响
 * @param <T>
 */
public class SelectSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        int minIndex = -1;
        int n = list.size();
        for (int i = 0; i < n - 1; i++) {
            minIndex = i;
            for (int j = i + 1; j < n; j++) {
                if (SortUtil.less(list.get(j), list.get(minIndex))) {
                    minIndex = j;
                }
            }
            if (SortUtil.equal(i, minIndex)) continue;
            SortUtil.swap(list, i, minIndex);
        }
    }
}

三、插入排序

插入排序（Insertion Sort）是一种简单但有效的排序算法，其基本思想是将待排序的元素逐个插入到已排序部分的合适位置。以下是插入排序的算法步骤：

  1. 初始时，将第一个元素视为已排序部分，其余元素为未排序部分。

  2. 从未排序部分中取出第一个元素，将其与已排序部分的元素逐个比较，找到合适的位置插入。

  3. 将选中的元素插入到已排序部分中的合适位置，此时已排序部分增加了一个元素，未排序部分减少了一个元素。

  4. 重复步骤2和步骤3，直到所有元素都被插入到已排序部分。

  5. 最终，整个序列按照要求的顺序排列完成。

插入排序的时间复杂度为$O(n^2)$，其中$n$是待排序元素的个数。插入排序在处理小型数据集时表现较好，因为它的内循环通常只需要少量比较和交换操作。然而，在大型数据集上性能较差，因此通常不适用于大规模排序任务。在实际应用中，对于小型数据集或部分已排序的数据集，插入排序可以是一个合理的选择。

插入排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 插入排序
 * 1. 如果数组长度为n，则需要比较n-1轮
 * 2. 核心思想是把一个元素插入到已经排好序的子数组中
 * 3. 第一次子数组的长度是1，数组长度是1，那么它必定是有序的数组
 * 4. 找到合适的插入位置，然后依次往后移动，以便空出位置，最后把元素插入到这个空出来的位置即可
 * @param <T>
 */
public class InsertSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        int n = list.size();
        for (int i = 1; i < n; i++) {
            T temp = list.get(i);
            int j = i - 1;
            // 寻找插入的位置
            while (j >= 0 && SortUtil.less(temp, list.get(j))) {
                list.set(j + 1, list.get(j));
                j--;
            }
            list.set(j + 1, temp);
        }
    }
}

四、折半插入排序

折半插入排序（Binary Insertion Sort）是一种改进的插入排序算法，它利用了二分查找来减少比较操作的次数。以下是折半插入排序的算法步骤：

  1. 初始时，将第一个元素视为已排序部分，其余元素为未排序部分。

  2. 从未排序部分中取出第一个元素，将其与已排序部分的元素逐个比较。与传统的插入排序不同，折半插入排序使用二分查找找到合适的插入位置。

  3. 使用二分查找来确定插入位置。首先，确定已排序部分的中间元素。如果当前元素小于中间元素，则插入位置在中间元素的左侧；否则，插入位置在中间元素的右侧。重复这个过程，直到确定插入位置。

  4. 将选中的元素插入到已排序部分中的合适位置，此时已排序部分增加了一个元素，未排序部分减少了一个元素。

  5. 重复步骤2和步骤3，直到所有元素都被插入到已排序部分。

  6. 最终，整个序列按照要求的顺序排列完成。

折半插入排序相对于普通插入排序的主要优势在于减少了比较的次数，从而在某些情况下能够提高性能。它的时间复杂度仍然为$O(n^2)$，但由于减少了比较操作，在某些情况下可能比普通插入排序更快。然而，与其他高效的排序算法相比，折半插入排序仍然不够快，通常适用于较小规模的数据集。

折半插入排序动图：

和插入排序的类似，只是插入排序是线性地查找待插入的位置，而折半插入排序是二分地查找待插入的位置，找到了之后也是一个个往后移动腾出位置，然后把数据插入进来。

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 折半插入排序
 * 插入排序的优化版
 * 优化的地方是普通的插入排序直接按顺序查找到要插入的位置
 * 而折半插入排序是先使用折半查找到要插入的位置，然后直接插入
 * 插入的方式和普通插入排序一样，都是找到位置之后就依次往后移动，然后空出位置就插入
 * @param <T>
 */
public class BinaryInsertSort<T extends Comparable<T>> implements MySort<T> {

    @Override
    public void sort(List<T> list) {
        int n = list.size();
        for (int i = 1; i < n; i++) {
            T temp = list.get(i);
            int low = 0;
            int high = i - 1;
            while (low <= high) {
                int mid = low + ((high - low) >> 1);
                if (SortUtil.less(temp, list.get(mid))) {
                    high = mid - 1;
                } else {
                    low = mid + 1;
                }
            }
            for (int j = i; j > low; j--) {
                list.set(j, list.get(j - 1));
            }
            list.set(low, temp);
        }
    }
}

五、希尔排序

希尔排序（Shell Sort）是一种改进的插入排序算法，它通过将序列分为多个子序列来进行排序，然后逐渐减少子序列的长度，最终完成排序。以下是希尔排序的算法步骤：

  1. 选择一个增量序列（gap sequence），这个序列是一系列递减的正整数，用于控制子序列的长度。通常，初始增量可以设为数组长度的一半，然后逐渐缩小增量。

  2. 根据选定的增量，将数组分成多个子序列，每个子序列包含间隔为增量的元素。例如，如果增量是3，那么第一个子序列包含第1、4、7、10等元素，第二个子序列包含第2、5、8、11等元素，以此类推。

  3. 对每个子序列应用插入排序算法，将子序列中的元素按照从小到大（或从大到小）的顺序排列。这里使用的是普通的插入排序，但是由于子序列长度较短，排序速度较快。

  4. 缩小增量，再次重复步骤2和步骤3，直到增量减小为1。当增量为1时，整个数组被看作一个子序列，继续进行插入排序。

  5. 最终，整个数组将按照要求的顺序排列完成。

希尔排序的性能取决于增量序列的选择，不同的增量序列可能导致不同的性能表现。希尔排序的时间复杂度通常介于$O(n^2)$和$O(n log n)$之间，具体取决于增量序列的选择。希尔排序相对于普通插入排序来说，在处理大型数据集时性能更好，但在实践中，通常会使用更高效的排序算法，如快速排序或归并排序。

希尔排序图解：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 希尔排序
 * 优化版的插入排序
 * 插入排序比较耗时间的其实是移动这块，插入排序只能一个一个移动
 * 而希尔排序是一组一组移动，每次移动一大步，最后一次移动的时候就插入排序
 * 所以希尔排序本质上就是插入排序，只是在移动元素上做了优化
 *
 * @param <T>
 */
public class ShellSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        int n = list.size();
        int h = 1;
        while (h < n / 3) {
            h = 3 * h + 1;
        }
        while (h >= 1) {
            for (int i = h; i < n; i++) {
                T temp = list.get(i);
                int j = i - h;
                while (j >= 0 && SortUtil.less(temp, list.get(j))) {
                    list.set(j + h, list.get(j));
                    j = j - h;
                }
                list.set(j + h, temp);
            }
            h = h / 3;
        }
    }
}

六、快速排序

快速排序（Quick Sort）是一种非常高效的排序算法，基于分而治之的思想。下面是算法步骤：

  1. 选择基准元素：从待排序的数组或列表中选择一个元素作为基准。选择方式有多种，如选择第一个元素、选择最后一个元素、选择中间元素或使用随机选择。这里选择第一个元素作为基准为例。

  2. 分区操作：代码进入一个循环，用两个指针i和j分别从数组的两端开始向中间移动，目的是将数组分为两部分，左侧部分包含所有小于基准元素的值，右侧部分包含所有大于基准元素的值。分区操作通过以下步骤实现：

    • 从右侧（高位，j指针）开始，寻找第一个小于基准元素的值，并将j指针向左移动，直到找到小于基准元素的值。
    • 从左侧（低位，i指针）开始，寻找第一个大于基准元素的值，并将i指针向右移动，直到找到大于基准元素的值。
    • 交换list.get(i)和list.get(j)，将找到的小于基准元素的值放在左侧，大于基准元素的值放在右侧。这一步将会使得小于基准元素的值在左侧紧邻基准元素，大于基准元素的值在右侧。

  3. 递归调用：分区操作完成后，基准元素已经位于正确的位置，左侧部分包含小于它的元素，右侧部分包含大于它的元素。接下来，递归调用quickSort函数对左侧和右侧的子数组进行排序。左侧子数组的范围是low到i-1，右侧子数组的范围是i+1到high。

  4. 终止条件：递归排序在low大于等于high时终止，此时表示子数组只包含一个元素或没有元素，无需进一步排序。

总体而言，上面描述的是经典的快速排序算法，通过不断地分区和递归排序，最终将整个数组按照升序排列完成。这是一个高效的排序算法，平均情况下具有$O(n log n)$的时间复杂度。

另一种快速排序动图：（本算法的动图没找到）

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

public class QuickSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        quickSort(list, 0, list.size() - 1);
    }

    private void quickSort(List<T> list, int low, int high) {
        if (low >= high) return;
        int i = low;
        int j = high;
        T pivot = list.get(i);
        while (i < j) {
            // 先从右边开始找，找到第一个小于pivot的值
            while (i < j && !SortUtil.less(list.get(j), pivot)) {
                j--;
            }
            list.set(i, list.get(j));
            while (i < j && !SortUtil.greater(list.get(i), pivot)) {
                ++i;
            }
            list.set(j, list.get(i));
        }
        list.set(i, pivot);
        quickSort(list, low, i - 1);
        quickSort(list, i + 1, high);
    }
}

七、归并排序

归并排序（Merge Sort）是一种高效的排序算法，它采用分而治之（Divide and Conquer）的策略来实现排序。以下是归并排序的算法步骤：

  1. 分割（Divide）：将待排序的数组或列表分成两个大致相等的子数组。这个过程一直递归，直到每个子数组只包含一个元素（已经是有序的）或没有元素。

  2. 合并（Merge）：将两个有序的子数组合并成一个有序的数组。合并操作是归并排序的核心，它将两个有序的子数组合并为一个有序的大数组。

    • 创建一个临时数组来存放合并后的结果。
    • 使用两个指针，分别指向两个子数组的起始位置。
    • 依次比较两个子数组中的元素，将较小（或较大，根据排序要求）的元素放入临时数组中，并将指针向前移动。
    • 重复上述比较和移动操作，直到其中一个子数组的元素全部被合并到临时数组中。
    • 将未合并完的子数组的剩余元素直接复制到临时数组中。
    • 最后，将临时数组的内容复制回原始数组，完成合并操作。

  3. 递归排序：对分割后的子数组重复上述步骤，即递归地对左半部分和右半部分进行分割和合并，直到整个数组都被合并排序为止。

  4. 终止条件：递归排序在每个子数组只包含一个元素或没有元素时终止。

归并排序的关键思想是将问题分解成小问题，然后逐步解决这些小问题，最终合并它们以得到最终的有序结果。它的时间复杂度是稳定的，为$O(n log n)$，其中$n$是待排序元素的个数。虽然归并排序在性能上可能不如快速排序那么快，但它的稳定性和可预测性使得它在需要稳定排序的情况下非常有用。

归并排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.ArrayList;
import java.util.List;

public class MergeSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        int n = list.size();
        List<T> temp = new ArrayList<>(n);
        temp.addAll(list);
        mergeSort(list, 0, n - 1, temp);
    }

    private void merge(List<T> list, int left, int mid, int right, List<T> temp) {
        int i = left;
        int j = mid + 1;
        int k = left;
        while (i <= mid && j <= right) {
            temp.set(k++, SortUtil.less(list.get(i), list.get(j))? list.get(i++) : list.get(j++));
        }
        while (i <= mid) {
            temp.set(k++, list.get(i++));
        }
        while (j <= right) {
            temp.set(k++, list.get(j++));
        }
        for (int x = left; x <= right; x++) {
            list.set(x, temp.get(x));
        }
    }

    private void mergeSort(List<T> list, int left, int right, List<T> temp) {
        if (left >= right) return;
        int mid = left + ((right - left) >> 1);
        mergeSort(list, left, mid, temp);
        mergeSort(list, mid + 1, right, temp);
        merge(list, left, mid, right, temp);
    }
}

八、堆排序

堆排序（Heap Sort）是一种基于二叉堆数据结构的排序算法，它利用堆的性质来进行排序。以下是堆排序的算法步骤：

  1. 建堆（Heapify）：将待排序的数组视为一个完全二叉树（通常是最大堆或最小堆），需要建立一个堆结构，使得数组满足堆的性质。建堆的过程可以分为两种方式：自底向上（Bottom-up）或自顶向下（Top-down）。通常采用自底向上的方式来建堆，因为它更高效。

    • 自底向上建堆：从数组的倒数第一个非叶子节点开始，逐个向前调整元素的位置，使得以当前节点为根的子树满足堆的性质。重复这个过程，直到整个数组满足堆的性质。

  2. 排序：建堆完成后，数组的第一个元素是堆中的最大（或最小）元素。将它与数组的最后一个元素交换位置，然后将堆的大小减1，即排除最后一个元素（最大或最小元素）。再次调整堆，使得剩余的元素满足堆的性质。

  3. 重复步骤2，直到所有元素都被排除。最终，整个数组就按照从小到大（或从大到小）的顺序排列完成。

堆排序的关键步骤是建堆和排序。建堆的时间复杂度是$O(n)$，排序的时间复杂度是$O(n log n)$，因此堆排序总体的时间复杂度是$O(n log n)$，其中$n$是待排序元素的个数。

堆排序的优点在于不需要额外的存储空间，因为它在原地进行排序，只需要一个额外的变量用于交换元素。然而，堆排序的常数因子较大，因此在实际应用中可能不如快速排序或归并排序那么快。堆排序通常用于需要稳定排序的情况下，或者当你需要同时进行排序和选择最大（或最小）值的操作时。

堆排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.List;

/**
 * 堆排序
 *  1. 把数据堆化
 *  2. 拿堆顶元素（最大值），放到最后，堆的个数减1
 *  3. 重复上面步骤，直到把所有元素都从堆中拿出来即可
 * @param <T>
 */
public class HeapSort<T extends Comparable<T>> implements MySort<T> {
    @Override
    public void sort(List<T> list) {
        int n = list.size();
        while (n > 0) {
            heapify(list, n);
            SortUtil.swap(list, 0, --n);
        }
    }

    /**
     * 堆化
     * @param list
     * @param n 当前堆的元素个数
     */
    private void heapify(List<T> list, int n) {
        int current = n / 2 - 1;
        while (current >= 0) {
            int i = current;
            int left = 2 * i + 1;
            while (left + 1 <= n) {
                int best = left + 1 < n && SortUtil.greater(list.get(left + 1), list.get(left)) ? left + 1 : left;
                if (SortUtil.greater(list.get(best), list.get(i))) {
                    SortUtil.swap(list, best, i);
                    i = best;
                    left = 2 * i + 1;
                } else {
                    break;
                }
            }
            current--;
        }
    }
}

九、计数排序

计数排序（Counting Sort）是一种非比较型的整数排序算法，适用于排序一定范围内的整数或具有确定范围的非负整数。计数排序的基本思想是统计每个元素在待排序序列中出现的次数，然后根据这些统计信息重建有序序列。以下是计数排序的算法步骤：

  1. 找出待排序序列中的最大值和最小值：首先遍历待排序序列，找到最大值和最小值，确定待排序元素的范围。

  2. 创建计数数组：创建一个计数数组（counting array），长度等于待排序元素范围的大小。计数数组用于存储每个元素出现的次数。

  3. 统计元素出现次数：遍历待排序序列，统计每个元素出现的次数，并将统计结果存储在计数数组中。具体步骤如下：

    • 遍历待排序序列的每个元素，将该元素作为计数数组的索引，将对应索引位置的计数值加1。

  4. 累加计数数组：对计数数组进行累加操作，将每个位置的值累加到它前面的位置上，得到一个累加计数数组。这个数组用于确定每个元素在有序序列中的位置。

  5. 构建有序序列：创建一个与待排序序列相同大小的输出数组（sorted array），用于存储排序后的结果。

    • 从待排序序列的末尾开始遍历，根据元素的值在累加计数数组中找到该元素在有序序列中的位置，并将它放入有序序列的对应位置。
    • 减少计数数组中对应元素的计数值，以便处理重复元素。

  6. 重复步骤5：重复步骤5，直到所有元素都被放入有序序列中。

  7. 最终，有序序列就是排序后的结果。

计数排序的时间复杂度为$O(n + k)$，其中$n$是待排序元素的个数，$k$是元素范围的大小。当k相对较小而且不远大于$n$时，计数排序是一个非常高效的排序算法。然而，计数排序不适用于范围非常大的整数或包含负数的情况。此外，它是稳定的排序算法，适用于需要保持相等元素相对顺序的情况。

计数排序动图：

代码：

package cn.cukor.sort;

import cn.cukor.sort.utils.SortUtil;

import java.util.ArrayList;
import java.util.List;

public class CountSort implements MySort<Integer> {
    private Integer low;
    private List<Integer> temp;

    public CountSort(Integer a, Integer b) {
        low = a;
        int n = b - a + 1;
        this.temp = new ArrayList<>();
        for (int i = 0; i < n; i++) {
            temp.add(0);
        }
    }

    @Override
    public void sort(List<Integer> list) {
        for (Integer v : list) {
            int index = v - low;
            temp.set(index, temp.get(index) + 1);
        }
        int n = temp.size();
        for (int i = 0, j = 0; i < n; i++) {
            while (SortUtil.greater(temp.get(i), 0)) {
                list.set(j++, i + low);
                temp.set(i, temp.get(i) - 1);
            }
        }
    }
}

十、基数排序

基数排序（Radix Sort）是一种非比较型的整数排序算法，它适用于对整数或其他可以按照位数进行分割的数据进行排序。基数排序的基本思想是按照低位到高位的顺序，依次对元素进行分配和收集，以达到排序的目的。以下是基数排序的算法步骤：

  1. 确定排序的位数：首先，确定待排序元素的最大位数（即最大值的位数）。这将决定需要进行多少轮分配和收集操作。

  2. 按位分配：从最低位（个位）开始，对待排序元素进行分配。具体步骤如下：

    • 创建10个桶（0到9），用于按照当前位的值将元素分配到桶中。每个桶对应一个位上的数字（0到9）。
    • 遍历待排序元素，根据当前位上的数字将元素放入相应的桶中。

  3. 收集：按照桶的顺序，依次将元素从桶中取出，形成一个新的排序后的序列。这一步的目的是将元素按照当前位的值从小到大重新排列。

  4. 重复分配和收集：依次对每一位进行分配和收集操作，从低位到高位，直到完成对所有位的排序。

  5. 最终，整个序列就按照要求的顺序排列完成。

需要注意的是，基数排序是一种稳定的排序算法，它的时间复杂度取决于待排序元素的位数和桶的数量。当待排序元素的位数较小且桶的数量合理时，基数排序可以达到线性时间复杂度，即O(n)。然而，在某些情况下，桶的数量可能会比较大，导致算法的时间复杂度增加。基数排序通常适用于整数或字符串等数据，但不适用于浮点数排序。

代码：

package cn.cukor.sort;

import java.util.ArrayList;
import java.util.List;

public class RadixSort implements MySort<Integer> {
    @Override
    public void sort(List<Integer> list) {
        List<List<Integer>> temp = new ArrayList<>();
        for (int i = 0; i < 10; i++) {
            temp.add(new ArrayList<>());
        }
        Integer max = getMax(list);
        Integer digits = getDigits(max);
        int k = 0;
        while (k < digits) {
            for (Integer num : list) {
                int index = num / (int)Math.pow(10, k) % 10;
                temp.get(index).add(num);
            }
            int j = 0;
            for (List<Integer> integers : temp) {
                for (Integer integer : integers) {
                    list.set(j++, integer);
                }
            }
            for (List<Integer> integers : temp) {
                integers.clear();
            }
            k++;
        }
    }

    private Integer getMax(List<Integer> list) {
        Integer max = list.get(0);
        int n = list.size();
        for (int i = 1; i < n; i++) {
            max = Math.max(list.get(i), max);
        }
        return max;
    }

    private Integer getDigits(Integer num) {
        if (num == 0) return 1;
        Integer digit = 0;
        while (num % 10 > 0) {
            digit++;
            num /= 10;
        }
        return digit;
    }
}

十一、桶排序（思想）

桶排序（Bucket Sort）是一种分布式排序算法，它的基本思想是将待排序的元素分到不同的桶（或箱子）中，然后分别对每个桶内的元素进行排序，最后按照桶的顺序和每个桶内的顺序将所有元素合并成有序序列。

以下是桶排序的关键思想和步骤：

  1. 分桶：将待排序的元素按照某种规则分配到不同的桶中。每个桶代表一定范围的元素值。通常，分桶的规则可以是将元素均匀地映射到桶中，以确保每个桶内的元素数量差不多。

  2. 排序：对每个非空桶内的元素进行排序。可以使用任何排序算法，通常选择插入排序或快速排序等。

  3. 合并：按照桶的顺序将所有非空桶内的元素合并起来，形成最终的有序序列。

与计数排序和基数排序的关系：

  • 计数排序：计数排序是一种特殊的桶排序，适用于待排序元素是非负整数且取值范围相对较小的情况。计数排序的基本思想是使用桶来统计每个元素的出现次数，然后按照元素的值的大小顺序将元素放入有序序列中。不同之处在于，计数排序的桶数量等于元素的取值范围大小，而桶排序通常可以根据元素的分布情况选择不同数量的桶。

  • 基数排序：基数排序也是一种特殊的桶排序，它适用于待排序元素是非负整数且每个元素有多位数表示的情况。基数排序的基本思想是从低位到高位依次使用桶对元素进行分配和收集，最终得到有序序列。基数排序是桶排序的一种扩展，它需要多次分配和收集操作，每次操作都是按照不同的位进行的。

总之，桶排序、计数排序和基数排序都是非比较型排序算法，它们的核心思想都是将元素分配到不同的桶中，然后对桶内的元素进行排序。它们的主要区别在于适用范围和分桶的规则。桶排序是一种通用的排序算法，可以适用于各种数据类型，而计数排序和基数排序通常有一定的限制条件。
