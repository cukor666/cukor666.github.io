git常用操作

git的介绍

版本控制工具，用于管理代码文件的版本。除了可以管理代码，也可以管理像word文档一样的文档，只是管理编程方面的代码，git可以很好的显示两个版本之间的不同。

git安装

直接到git官网 下载并安装即可，安装的时候一路下一步，没有什么难度。

设置git信息

git安装完成之后，第一件事情就是去设置git的信息，这个设置只需要设置一次即可，后面基本上用不到了。这个设置的关键就在于邮箱，虽然此时的git的邮箱设置并不会真正去检验这个邮箱是否存在，但是后续会和github或者是gitee结合起来，所以这里设置的邮箱最好就是github或gitee绑定的邮箱，防止到时候又出现问题，在一开始的时候直接设置好，后面就不会出现邮箱不存在的问题。

git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

如果用了 –global 选项，那么更改的配置文件就是位于你用户主目录下的那个，以后你所有的项目都会默认使用这里配置的用户信息。

如果要在某个特定的项目中使用其他名字或者电邮，只要去掉 –global 选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

设置完成之后可以通过下面的命令查看刚刚设置的信息。

git config --list

体验第一次git

接下来去实操一下git。

在你的电脑中找到一个空的目录（最好不要出现中文，因为可能会出现中文乱码，虽然问题也不大，但是看着不爽）。最好使用git自带的git bash终端，不要使用windows下的其他命令行工具。

初始化仓库

Git 使用 git init 命令来初始化一个 Git 仓库，Git 的很多命令都需要在 Git 的仓库中运行，所以 git init 是使用 Git 的第一个命令。

在执行完成 git init 命令后，Git 仓库会生成一个 .git 目录（这个目录是隐藏的），该目录包含了资源的所有元数据，其他的项目目录保持不变。

使用当前目录作为 Git 仓库，我们只需使它初始化。

git init

该命令执行完后会在当前目录生成一个 .git 目录。

使用我们指定目录作为Git仓库。

git init newrepo

初始化后，会在 newrepo 目录下会出现一个名为 .git 的目录，所有 Git 需要的数据和资源都存放在这个目录中。

那么，现在我们来到/d/myCoding/study/git-test目录下，执行git init.

这个结果表示初始化git仓库成功，并且生成了一个.git文件夹。这个文件夹是隐藏的文件夹，Windows下需要把隐藏文件的设置打开才可以看得到。

那么到这一步，初始化仓库的工作就已经完成了。

添加文件

添加下面的文件，用于测试。

Hello.java：

public class Hello {
    public static void main(String[] args) {
        System.out.println("HelloWorld");
    }
}

添加文件到暂存区

想要使用git来管理文件，就必须先使用git add 文件把文件添加到暂存区。然后可以使用git status来查看当前git管理文件的状态。

那么现在Hello.java文件已经放到暂存区中了。

提交

现在Hello.java只是添加到了暂存区，git还不能管理到Hello.java文件，还需要把Hello.java提交到本地仓库，git才可以管理到它。

提交的命令是：git commit -m '提交时的信息或备注'。

那么现在Hello.java文件已经提交到本地仓库中了，git就已经可以管理这个Hello.java文件。

修改文件

将Hello.java修改为如下内容：

public class Hello {
    public static void main(String[] args) {
        System.out.println("你好世界");
    }
}

仅仅只是把英文输出换成了中文输出。

查看当前git的状态。

这个说明，我们已经把Hello.java的内容做了修改，但是还没有提交到暂存区，它这里提示我们可以把它提交到暂存区。

再次添加文件到暂存区

此时，暂存区中的Hello.java和本地仓库中的Hello.java是不一样的。

查看差异

我们可以通过git diff HEAD来查看暂存区中的Hello.java和本地仓库中的Hello.java的差异。

可以看到它们之间的差异。红色的表示原来的（本地仓库中），绿色的表示现在的（暂存区中）。

如果我们想使用现在（暂存区中）的，那就执行git commit操作，把它提交到本地仓管，这样就替换了原来的文件。

如果我们想使用原来（本地仓库中）的，那就执行git rm操作，把暂存区中的文件删除掉（从暂存区中移除出来）。

从暂存区移除

我们现在就想使用原来本地仓库中的，而不想使用目前暂存区的文件。

  • -f操作表示强制删除

然后会发现，我们本地的文件夹下的Hello.java也被删除掉了。

所以git rm操作，其实是将文件从暂存区和工作区中删除。

再次提交

因为刚刚已经把本地文件夹下的Hello.java已经删除掉了，而现在我们想把下面的文件替换掉本地仓库的文件。

public class Hello {
    public static void main(String[] args) {
        System.out.println("你好世界");
        System.out.println("你好git");
    }
}

所涉及到的命令之前已经用过了，这里不再赘述，直接干。

回退版本

刚刚已经再次提交了，但是我现在又不想使用当前版本的Hello.java，我还是想要之前的，但是本地仓库中的Hello.java已经被替换成最新的了，有没有什么办法可以回退到上一个版本。

git reset命令用于回退版本，可以指定退回某一次提交的版本。

使用方法：

git reset HEAD^            # 回退所有内容到上一个版本  
git reset HEAD^ Hello.java # 回退 Hello.java 文件的版本到上一个版本  
git reset  052e            # 回退到指定版本

前两种方法都好理解，第三种方法就必须拿到版本的版本好可以。

那这里展示第三种方式：

  • –hard 参数撤销工作区中所有未提交的修改内容，将暂存区与工作区都回到上一次版本，并删除之前的所有信息提交。
  • 注意：谨慎使用 –-hard 参数，它会删除回退点之前的所有信息。

现在已经回到了初次提交的版本。对应本地的Hello.java文件已经自动变成最原始的版本了。

这个是自动变的，不是人为修改的Hello.java。

体验第一次git结束

那么跟着上面的操作下来之后，你应该是掌握了基本的git的命令了。并且这个是本地仓库而已，后续会用到像github和gitee这样的远程仓库。但在此之前，我们先来介绍一个新的软件——GitHub Desktop。

图形化界面GitHub Desktop

命令行的git操作起来比较有门槛，而且不易观察。因此出现了图形化界面的GitHub Desktop。GitHub Desktop其实也是基于命令行，但是它操作起来就比命令行简单很多倍，并且以美观的界面展示出来，让开发者能直接一眼就看明白，可谓是神器。

软件的下载和安装，直接到官网下载即可，然后安装也是一路下一步就搞定了。默认是英文版的，如果想要中文版，也可以在github上找到对应的的中文包，直接执行就可以把GitHub Desktop变成中文版。

GitHub Desktop：desktop.github.com

中文包：robotze/GithubDesktopZhTool: Github Desktop 汉化工具 支持 Windows Mac Linux

体验GitHub Desktop

添加Person.java和修改Hello.java。

这个界面体验感可比命令行强多了。至少看着没有那么压抑。

填写提交信息后，直接按下提交按钮就可以提交了。

这边还可以直接看到提交的历史。

GitHub Desktop还可以一键直接提交到github上。

但可能会因为网络的问题，导致发布不成功，这个时候，懂的都懂，不懂的也没必要了解了。

如果成功发布到github上之后，从github上直接拉取到本地也是很简单的事情。点点点就完事了。

然后GitHub Desktop也不仅仅是拉取github，同时也可以拉取gitee上的仓库，因为它提供了URL拉取的方式，只需要把gitee上对应仓库的URL复制过来就可以拉取到了，然后也可以直接在本地和gitee做同步连接操作。

对接github

GitHub Desktop对接github是非常简单的，不需要在github上预先创建仓库，直接通过GitHub Desktop直接发布到github即可。

这样就已经发布到github上了，仓库自己就创建好了，不需要我们再到github上进行创建。

这个只是远程仓库修改了，但是本地却没有修改，所以需要在本地拉取远程仓库中的内容，这样就同步了。在GitHub Desktop是很简单的。

对接gitee

github毕竟是国外的，所以用起来网络会有点让人不爽。那么gitee其实算是github的国内版。我们在国内用就不存在网络特别卡的问题。

好，这样就已经发布到gitee中了，然后我们回到网页端，看看我们的东西是不是已经发布到gitee上了。

IDEA配置git

IDEA中默认就配置好github，但是需要登录github账号，然后做这个配置的过程中，容易出现网络异常的问题导致IDEA无法把本地仓库发布到远程仓库中，比较麻烦，操作步骤都算是简单的，没有特别复杂，但是就是容易败给网络。那既然是这样，我们干脆就直接使用IDEA结合gitee好了。这样的话速度是比较快的。

但是IDEA本身没有默认配置gitee，需要我们人为安装Gitee插件，然后才可以在IDEA中对接gitee。
