阅读Go源码——container/list

Go语言提供了一个容器——list。

list的底层是实现的双向链表。这个双向链表是带有头节点的，并且Go官方给它设置的属性名叫root.

并且告诉我们list的遍历可以使用下面的方式遍历：

for e := l.Front(); e != nil; e = e.Next() {
	// do something with e.Value
}

它没有像C++或Java中的迭代器的方式遍历，但其实C++或Java的迭代器遍历底层也是通过指针或者引用来遍历的。Go语言这里的设计就比较简洁了。

链表节点的结构

下面这个是官方的源码。

// Element is an element of a linked list.
type Element struct {
	// Next and previous pointers in the doubly-linked list of elements.
	// To simplify the implementation, internally a list l is implemented
	// as a ring, such that &l.root is both the next element of the last
	// list element (l.Back()) and the previous element of the first list
	// element (l.Front()).
	next, prev *Element

	// The list to which this element belongs.
	list *List

	// The value stored with this element.
	Value any
}

那么可以通过官方的注释知道，Element其实就是我们在数据结构课堂上学习到的节点的结构体描述。

Element属性介绍：

  • next：当前节点的下一个节点的指针
  • prev：当前节点的上一个节点的指针
  • list：当前节点归属的链表，如果当前节点没有归属的链表则为nil
  • Value：当前节点存储的实际数据，这里是any类型，就是interface{}，表示可以存放任意数据类型

链表节点的方法

只有两个方法，分别是：

  • Next() *Element：获取当前节点的下一个节点的指针
  • Prev() *Element：获取当前节点的上一个节点的指针
// Next returns the next list element or nil.
func (e *Element) Next() *Element {
	if p := e.next; e.list != nil && p != &e.list.root {
		return p
	}
	return nil
}

// Prev returns the previous list element or nil.
func (e *Element) Prev() *Element {
	if p := e.prev; e.list != nil && p != &e.list.root {
		return p
	}
	return nil
}

其中e.list != nil就表示了当前节点一定要是某个链表的节点，否则就没有下一个节点和上一个节点之说了，直接返回nil.

并且p不应该是当前链表的root，因为root不存放节点的数据，root是整个链表的控制中心。

链表的结构

Go语言对节点再次封装，把节点和链表分开来将，并且在链表中添加了一个Element类型的控制中心节点root，它只负责控制整个链表，而不存放数据。

// List represents a doubly linked list.
// The zero value for List is an empty list ready to use.
type List struct {
	root Element // sentinel list element, only &root, root.prev, and root.next are used
	len  int     // current list length excluding (this) sentinel element
}

Go语言还为list封装了一个len的属性，这样可以直接在内部快速拿到整个链表的大小，这样就不需要每次需要获取链表的大小的时候还要去遍历整个链表。并且root和len都是当前包可以见，其他包不可以，这样做请确保了安全性，不允许其他地方擅自修改。

链表的方法

简单方法

获取链表大小

当然其他包有可能也会需要获取链表的大小，那么Go也提供了方法获取，但是只能获取读的权力，不能修改：

// Len returns the number of elements of list l.
// The complexity is O(1).
func (l *List) Len() int { return l.len }

获取头部（第一个带有数据的节点）

// Front returns the first element of list l or nil if the list is empty.
func (l *List) Front() *Element {
    if l.len == 0 {
       return nil
    }
    return l.root.next
}

获取尾部（最后一个带有数据的节点）

// Back returns the last element of list l or nil if the list is empty.
func (l *List) Back() *Element {
    if l.len == 0 {
       return nil
    }
    return l.root.prev
}

初始化

Init

// Init initializes or clears list l.
func (l *List) Init() *List {
	l.root.next = &l.root
	l.root.prev = &l.root
	l.len = 0
	return l
}

最本质的方法，其他变种都是内部调用了它。

一开始next和prev都指向了root本身，因为root不存放数据，所以这样操作还OK。

  • root.next：表示了链表中的第一个存放数据的节点。
  • root.prev：表示了链表中的最后一个存放数据的节点。

New

// New returns an initialized list.
func New() *List { return new(List).Init() }

Init的封装。区别在于Init是通过对象调用，而New是函数，不需要通过对象来调用，直接调用即可。

lazyInit

// lazyInit lazily initializes a zero List value.
func (l *List) lazyInit() {
	if l.root.next == nil {
		l.Init()
	}
}

小写开头，说明外界（其他包）是访问不到的。它的功能就是如果对象如果还没有初始化就会自动初始化，如果已经初始化了，就不需要再次初始化了，这个方法在添加的功能里就有调用，例如，当我们的list对象只是声明了，但是并没有做初始化操作，也可以直接调用插入添加的方法，因为这些添加插入的方法里面就调用了这个lazyInit方法。

添加/插入

insert：本质

// insert inserts e after at, increments l.len, and returns e.
func (l *List) insert(e, at *Element) *Element {
    e.prev = at
    e.next = at.next
    e.prev.next = e
    e.next.prev = e
    e.list = l
    l.len++
    return e
}

这个方法直接通过已知的节点的地址at直接添加，就类似数组直接通过索引取值一样那么快。

insertValue

// insertValue is a convenience wrapper for insert(&Element{Value: v}, at).
func (l *List) insertValue(v any, at *Element) *Element {
    return l.insert(&Element{Value: v}, at)
}

insert方法的封装。

PushFront

// PushFront inserts a new element e with value v at the front of list l and returns e.
func (l *List) PushFront(v any) *Element {
    l.lazyInit()
    return l.insertValue(v, &l.root)
}

插入数据到链表的头部（第一个带有数据的节点）。

这里就调用了lazyInit，就是防止程序员还没有对链表进行初始化就插入数据的错误，所以这里调用了它就解决了这个问题，如果没有初始化它会帮我们初始化。

因为前面的插入方法已经写好了，这里直接调用已经写好的方法即可。既然是插入到头部，那就放到root的后面，那传参的时候直接传入&l.root即可。

PushBack

// PushBack inserts a new element e with value v at the back of list l and returns e.
func (l *List) PushBack(v any) *Element {
    l.lazyInit()
    return l.insertValue(v, l.root.prev)
}

和PushFront类似，既然要插入到链表的尾部（最后一个带有数据的节点），那参数就传l.root.prev，因为它就表示链表的最后一个节点的地址。

InsertBefore

// InsertBefore inserts a new element e with value v immediately before mark and returns e.
// If mark is not an element of l, the list is not modified.
// The mark must not be nil.
func (l *List) InsertBefore(v any, mark *Element) *Element {
    if mark.list != l {
       return nil
    }
    // see comment in List.Remove about initialization of l
    return l.insertValue(v, mark.prev)
}

这个方法的本质也是调用了已经写好的方法：insertValue，只需要确认节点的地址即可。这里多了一个是判断标记的节点是否是当前链表中，如果是在当前链表中才做插入的操作，否则就返回nil不做插入的操作.

InsertAfter

// InsertAfter inserts a new element e with value v immediately after mark and returns e.
// If mark is not an element of l, the list is not modified.
// The mark must not be nil.
func (l *List) InsertAfter(v any, mark *Element) *Element {
    if mark.list != l {
       return nil
    }
    // see comment in List.Remove about initialization of l
    return l.insertValue(v, mark)
}

和前面类似，只需要确定插入的位置。注意是在之前还是之后。

添加其他链表

// PushBackList inserts a copy of another list at the back of list l.
// The lists l and other may be the same. They must not be nil.
func (l *List) PushBackList(other *List) {
    l.lazyInit()
    for i, e := other.Len(), other.Front(); i > 0; i, e = i-1, e.Next() {
       l.insertValue(e.Value, l.root.prev)
    }
}

// PushFrontList inserts a copy of another list at the front of list l.
// The lists l and other may be the same. They must not be nil.
func (l *List) PushFrontList(other *List) {
    l.lazyInit()
    for i, e := other.Len(), other.Back(); i > 0; i, e = i-1, e.Prev() {
       l.insertValue(e.Value, &l.root)
    }
}

其实就是遍历其他链表other，把它的数据按头插入方式放到当前链表中，或者按尾插入方式放到当前链表中。

删除

remove：本质

// remove removes e from its list, decrements l.len
func (l *List) remove(e *Element) {
    e.prev.next = e.next
    e.next.prev = e.prev
    e.next = nil // avoid memory leaks
    e.prev = nil // avoid memory leaks
    e.list = nil
    l.len--
}

和插入的操作一样直接获取节点的地址，这样不用遍历。

Remove

// Remove removes e from l if e is an element of list l.
// It returns the element value e.Value.
// The element must not be nil.
func (l *List) Remove(e *Element) any {
    if e.list == l {
       // if e.list == l, l must have been initialized when e was inserted
       // in l or l == nil (e is a zero Element) and l.remove will crash
       l.remove(e)
    }
    return e.Value
}

对remove方法的封装，这个Remove是给外部调用的，所以需要判断当前要删除的节点e是否是当前链表中，如果是才做删除。

移动

move：本质

// move moves e to next to at.
func (l *List) move(e, at *Element) {
    if e == at {
       return
    }
    e.prev.next = e.next
    e.next.prev = e.prev

    e.prev = at
    e.next = at.next
    e.prev.next = e
    e.next.prev = e
}

链表的移动就要比数组的移动要简单快捷很多了。

这个方法是将链表中的e节点移动到at节点后面，同样也是直接通过指针来操作。

如果e就是at本身，那就不需要进行移动的操作了。

移动的操作其实就是先把e删除出来，然后再插入到at后面，这样就实现了移动的效果了。

MoveToFront

// MoveToFront moves element e to the front of list l.
// If e is not an element of l, the list is not modified.
// The element must not be nil.
func (l *List) MoveToFront(e *Element) {
    if e.list != l || l.root.next == e {
       return
    }
    // see comment in List.Remove about initialization of l
    l.move(e, &l.root)
}

移动到头部（第一个带有数据的节点）。那就是root的后面。前面已经写好了move，这里就只需要调整一下参数即可。

MoveToBack

// MoveToBack moves element e to the back of list l.
// If e is not an element of l, the list is not modified.
// The element must not be nil.
func (l *List) MoveToBack(e *Element) {
    if e.list != l || l.root.prev == e {
       return
    }
    // see comment in List.Remove about initialization of l
    l.move(e, l.root.prev)
}

类似MoveToFront，只不过这个方法是移动到尾部。

MoveBefore和MoveAfter

// MoveBefore moves element e to its new position before mark.
// If e or mark is not an element of l, or e == mark, the list is not modified.
// The element and mark must not be nil.
func (l *List) MoveBefore(e, mark *Element) {
    if e.list != l || e == mark || mark.list != l {
       return
    }
    l.move(e, mark.prev)
}

// MoveAfter moves element e to its new position after mark.
// If e or mark is not an element of l, or e == mark, the list is not modified.
// The element and mark must not be nil.
func (l *List) MoveAfter(e, mark *Element) {
    if e.list != l || e == mark || mark.list != l {
       return
    }
    l.move(e, mark)
}

这几个方法都是类似的，只不过是在move方法上做了参数的调整而已。这里就不在赘述了。

简单使用

package main

import (
    "container/list"
    "fmt"
)

func main() {
    l := list.New()
    l.PushBack(3)
    p := l.PushBack(2)
    l.PushBack(1)
    for e := l.Front(); e != nil; e = e.Next() {
       fmt.Print(e.Value, " ")
    }
    l.InsertAfter(55, p)

    fmt.Printf("\n-----------------------------------\n")

    var p1 *list.Element = nil
    var p2 *list.Element = nil
    for e := l.Front(); e != nil; e = e.Next() {
       fmt.Print(e.Value, " ")
       if e.Value == 1 {
          p1 = e
       }
       if e.Value == 2 {
          p2 = e
       }
    }

    l.MoveBefore(p1, p2)

    fmt.Printf("\n-----------------------------------\n")

    for e := l.Front(); e != nil; e = e.Next() {
       fmt.Print(e.Value, " ")
    }
}

结果：

3 2 1
-----------------------------------
3 2 55 1
-----------------------------------
3 1 2 55
