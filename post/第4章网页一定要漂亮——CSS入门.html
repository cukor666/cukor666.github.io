第4章 网页一定要漂亮——CSS入门

上一章HTML制作的网页有点丑。HTML可以美化网页，只是它的代价是以臃肿的代码和延迟的下载速度为前提。为了更高效地开发网页，美化网页的任务一般会交给专用的技术——CSS。它是一种让网页变得更漂亮的技术。本章主要介绍CSS的作用、工作原理、基本用法以及常用样式的定义方法。

4.1 为什么需要CSS

为了让网页更好看，让网页加载更快。

4.1.1 什么是CSS

CSS（Cascading Style Sheets，层叠样式表），是一种样式语言，用于修饰HTML设计的元素，而且告诉浏览器如何将这些元素精美地显示在网页中。

为了更好地理解CSS的准确含义，让我们按字面意思从后往前看。首先，一个网页的外观控制需要定义一系列的样式规则，这些规则组成了一个样式表。其次，由于元素的嵌套关系会带来样式的继承或直接为了一个元素规定多种样式，导致一个元素通常会受到外层样式规则的影响，这时就形成了在显示层级、显示顺序和显示优先级方面的层叠关系，最终浏览器只选择其中一种样式显示元素。至此，CSS的名字便一目了然，不过，为了便于记忆，你可以简称它为CSS或样式即可。

4.1.2 CSS工作原理

CSS的本质就是定义元素的样式规则，告诉浏览器如何在网页中显示元素；比如，将标题颜色设置为红色。那么HTML、CSS和浏览器到底要怎么配合才能实现这个过程？我们已经知道各浏览器都必须遵守W3C的约定，才能做到正确地解析HTML的各类标签，那么对于CSS它又该如何解释呢？其实，浏览器首先要解析HTML，并且创建一颗DOM树，然后下载并解析CSS，目的是找到元素的样式定义。这样，HTML和CSS结合便能够画出一颗文档对象模型树（简称DOM树），这里的DOM其实是计算机上存储的一个文档，它保存了文档内容和样式的逻辑关系，浏览器最终根据DOM树绘制并显示网页的元素，主要过程如下图所示：

为了理解DOM树，我们来简单看一段添加了CSS样式的代码片段：

<!DOCTYPE html>
<html>

<head>
    <meta charset="UTF-8">
    <title>Document</title>
    <style>
        p {
            color: green;
            font-size: 16px;
        }
    </style>
</head>

<body>
    <p>HelloWorld</p>
</body>

</html>

它对应的DOM树如下图所示：

其中，每一个不同的形状都是一个节点，节点的类型采用不同的形状表示。需要说明的是，这里我们只绘制了一棵DOM树，但如果CSS样式是被定义在一个独立的外部文件中，那么真实的DOM树会是两棵，一棵是HTML对应的DOM树，一棵是CSS对应的DOM树。之所以CSS文档也会解析成树的结构，是方便浏览器去检查顶部的通用规则，比如应用于body的样式，然后再逐级向下找到某一个精确元素的样式，如果发现有重复定义的样式，则越靠后的样式定义会覆盖之前的样式，所以就有了层叠样式的由来。

什么是DOM树？

DOM树实际上是对文档的一种逻辑结构抽象，人们想象着HTML元素和CSS样式的层级结构就像一棵树根被倒置过来，有树根<document>、无数树枝（<html>、<head>、<body>）以及叶子组成等，而每一树枝和叶子都叫作一个节点，于是，文档中的所有代码在计算机中采用树的数据结构进行存储。这里节点可以是一个元素，一个属性，甚至是一段文本等。关于文档中的层级关系，是通过树枝与叶子节点之间的关系来实现的，因此会出现一些父亲节点，另一些是儿子节点，拥有同一个父亲的节点则是兄弟关系。

4.1.3 CSS带来了哪些变化

网页中有无CSS还是区别很大的。简单来说，HTML只提供了网页的框架结构和内容，样式则是对内容的修饰，包括文字的颜色变化、内容和布局、列表的图标设置等，总之有了CSS，你的HTML页面才会变得更出众。

这是没有使用CSS样式修饰的HTML，源代码如下：

<!DOCTYPE html>
<html>
<head>
    <title>我的网页</title>
</head>
<body>
    <h1>欢迎来到我的网页</h1>
    <p>这是一个段落。</p>
    <a href="https://www.bing.com">这是一个指向Bing搜索引擎的超链接</a>
    <ul>
        <li>列表项1</li>
        <li>列表项2</li>
        <li>列表项3</li>
    </ul>
</body>
</html>

4.2 容器的作用

在介绍更多的CSS样式之前，我们先来看两个特别重要的标签，它们在HTML中出现的频率非常高。

容器，顾名思义，就是用来存放物品的。在网页中，容器就是用来设置网页内容的，但不同之处是它自身是不可见的。就是说，如果容器中不放任何内容，你根本意识不到它的存在。但是，它又是存在的，它存在是以一种块标记的格式出现，也就是说它规定了它内部的标签内容与前后元素之间有一定的间距。

本节介绍两个特殊的容器标签：

  • div
  • span

<div>和<span>都是HTML中的通用元素，用于将网页的相关部分分组在一起。但是，它们的功能不同。<div>元素用于块级组织和样式化页面元素，而<span>元素用于内联组织和样式化。

<div>是一个块级元素，通常用于关联和分组网页的较大部分，例如页眉、页脚、主要内容等。下面是一个使用<div>标签将网页的页眉分组在一起并使用CSS进行样式化的示例：

<div class="header">
    <h2 class="logo">freeCodeCamp</h2>
    <ul class="nav">
        <li><a href="">Home</a></li>
        <li><a href="">About</a></li>
        <li><a href="">Services</a></li>
        <li><a href="">Contact</a></li>
    </ul>
</div>
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
.header {
    padding: 0 70px;
    display: flex;
    align-content: center;
    justify-content: space-between;
    margin-top: 20px;
    margin-bottom: 20px;
    color: crimson;
}
.nav {
    display: flex;
    align-content: center;
    justify-content: center;
    gap: 60px;
    list-style-type: none;
}
.nav li a {
    text-decoration: none;
    font-size: 1.2rem;
    color: crimson;
}

另外，您可以使用<div>标签将类似的内容分组在一起。这可以是类似的文本、图像、视频等。因此，您可以在<div>中嵌套<div>，并为它们附加唯一的类或id属性，以免混淆。

<span>是一个内联元素，用于使内容的较小部分通过CSS或JavaScript突出显示。您不应嵌套<span>，除非您非常清楚自己在做什么，但您可以在块级元素内放置多个<span>标签。下面是一个示例，我通过在它们周围包装<span>标签并使用不同的样式使某些特定单词突出显示：

<p>
This is a <span class="crimson">crimson text</span> within black texts.
This is an <span class="indigo">indigo text</span> within others,
and this is an <span class="orange">orange text</span> within other texts.
</p>
body {
    display: flex;
    align-items: center;
    justify-content: center;
    max-width: 900px;
    margin: 0 auto;
    height: 100vh;
}
p {
    font-size: 2.5rem;
}
.font-style {
    font-style: italic;
}
.crimson {
    color: crimson;
}
.indigo {
    color: indigo;
}
.orange {
    color: orange;
}

4.3 四种定义样式的方式

有四种定义CSS样式的方式：

方式描述
内联样式在HTML元素内使用style属性
内部样式在HTML文档的<head>部分内使用<style>元素
外部样式使用<link>元素链接到外部CSS文件
导入样式在CSS文件中使用@import规则导入其他CSS文件

最常见的添加CSS的方式是将样式保存在外部CSS文件中。

这里是四种定义CSS样式的方式的示例：

  1. 内联样式：在HTML元素内使用style属性。例如：

    <p style="color: red;">这是一段红色文本。</p>

  2. 内部样式：在HTML文档的<head>部分内使用<style>元素。例如：

    <head>
    <style>
        p {
            color: red;
        }
    </style>
</head>
<body>
    <p>这是一段红色文本。</p>
</body>

  3. 外部样式：使用<link>元素链接到外部CSS文件。例如，在HTML文档中：

    <head>
    <link rel="stylesheet" href="styles.css">
</head>
<body>
    <p>这是一段红色文本。</p>
</body>

    在styles.css文件中：

    p {
    color: red;
}

  4. 导入样式：在CSS文件中使用@import规则导入其他CSS文件。例如，在styles.css文件中：

    @import url('other-styles.css');

p {
    color: red;
}

    在other-styles.css文件中：

    body {
    background-color: lightblue;
}

最常用的方式是外部样式。它允许您将所有样式保存在一个单独的CSS文件中，并在多个HTML页面中使用。这样，您可以通过更改一个文件来更改整个网站的外观。这种方式可以使代码更加整洁、易于维护，并且可以提高页面加载速度。

4.3.1 样式优先级

以上这四种定义CSS样式的方式存在优先级关系。它们的优先级顺序如下：

  1. 内联样式：优先级最高。
  2. 内部样式：优先级次之。
  3. 外部样式：优先级最低。

如果同一个HTML元素被多个CSS规则所匹配，浏览器将根据优先级应用其中一个值。例如，如果一个元素同时具有内联样式和内部样式，那么内联样式将覆盖内部样式。

此外，如果两个或多个规则具有相同的优先级，则后定义的规则将覆盖先定义的规则。例如，如果您在CSS文件中定义了两个相同选择器的规则，那么后面的规则将覆盖前面的规则。

4.3.2 样式不起作用的检查方案

如果您的CSS样式不起作用，您可以采取以下步骤来检查它：

  1. 检查CSS文件的路径是否正确。您可以在浏览器中右键单击页面并单击“查看源代码”，然后单击style.css，看看它是否打开了您的CSS文件。如果没有，那么您设置的路径可能是错误的。

  2. 检查CSS选择器是否正确。如果您的CSS文件已成功加载，但样式仍然不起作用，那么您可能需要检查您的CSS选择器是否正确。

  3. 使用浏览器的开发者工具。大多数现代浏览器都提供了开发者工具，可以帮助您检查和调试CSS问题。例如，在Google Chrome中，您可以按F12键打开开发者工具，然后在“元素”选项卡下检查应用于特定元素的CSS样式。

  4. 清除浏览器缓存。有时，浏览器缓存可能会导致CSS样式不起作用。您可以尝试清除浏览器缓存并刷新页面，看看问题是否得到解决。

4.4 盒子模型

4.4.1 盒子模型的基本定义

盒子模型是前端开发中一个重要的概念。它定义了HTML元素如何在页面上呈现，包括元素的内容、内边距、边框和外边距。

在盒子模型中，每个HTML元素都可以看作一个矩形盒子，由四个部分组成：

  1. 内容（Content）：元素的实际内容，例如文本、图像等。
  2. 内边距（Padding）：内容与边框之间的空间。
  3. 边框（Border）：围绕元素的线条。
  4. 外边距（Margin）：元素与其他元素之间的空间。

您可以使用CSS来控制这些部分的大小和样式。例如，您可以使用padding属性来控制内边距的大小，使用border属性来控制边框的样式和大小，使用margin属性来控制外边距的大小。

了解盒子模型对于创建复杂的页面布局非常重要。它可以帮助您更好地控制元素在页面上的位置和外观。

关于width和height的“坑”

在给某个元素定义宽度和高度属性时，是指盒子内部的内容的宽度和高度，而不是盒子的实际值。盒子的实际高度和宽度的计算方式如下：

盒子宽度=border-left + padding-left + width + padding-right + border-right

盒子高度=border-top + padding-top + height + padding-bottom + border-bottom

4.5 选择器

在CSS中，选择器用于选择要应用样式的HTML元素。有许多不同类型的选择器，它们可以根据元素的标签、类、ID、属性等来选择元素。

下面是一些常见的CSS选择器及其示例：

  1. 元素选择器：根据元素的标签名称选择元素。例如，p选择器将选择所有<p>元素。

    p {
    color: red;
}

  2. 类选择器：根据元素的类属性选择元素。例如，.highlight选择器将选择所有具有class="highlight"属性的元素。

    .highlight {
    background-color: yellow;
}

  3. ID选择器：根据元素的ID属性选择元素。例如，#header选择器将选择具有id="header"属性的元素。

    #header {
    font-size: 24px;
}

  4. 属性选择器：根据元素的属性和值选择元素。例如，[href]选择器将选择所有具有href属性的元素，而[href="https://www.example.com"]选择器将仅选择具有href="https://www.example.com"属性的元素。

    [href] {
    color: blue;
}
[href="https://www.example.com"] {
    font-weight: bold;
}

除了这些基本的选择器，CSS还提供了许多其他类型的选择器，可以更精确地控制要应用样式的元素。您可以在W3Schools或MDN上查看完整的CSS选择器参考。

4.5.1 选择器的优先级关系

CSS选择器之间存在优先级关系。当多个选择器匹配同一个元素时，具有最高优先级的选择器将“获胜”，并将其样式应用于该元素。

选择器的优先级是根据其组成部分计算的。每种选择器类型都有一个特定的权重值，这些值相加得到选择器的总优先级。权重值按以下顺序递减：

  1. 内联样式：权重值为1000。
  2. ID选择器：每个ID选择器的权重值为100。
  3. 类选择器、属性选择器和伪类：每个类选择器、属性选择器或伪类的权重值为10。
  4. 元素选择器和伪元素：每个元素选择器或伪元素的权重值为1。

例如，#header选择器的优先级为100，而.highlight选择器的优先级为10。因此，如果一个元素同时具有id="header"和class="highlight"属性，则#header选择器将“获胜”，并将其样式应用于该元素。

此外，如果两个或多个规则具有相同的优先级，则后定义的规则将覆盖先定义的规则。例如，如果您在CSS文件中定义了两个相同选择器的规则，那么后面的规则将覆盖前面的规则。
