环境搭建

前言

数据结构与算法之前已经有过笔记，并且视频也放在B站上，我开始涨粉也是从《十大经典排序算法（C语言版）》的时候开始。但我当时并没有及时地把笔记发布出来，算是错过了一些良机，后面我把它的笔记发布到博客园的时候却没有人知道。

《十大经典排序算法（C语言版）》笔记

然后简单总结一下大家反馈给我的问题：

  1. 这个名字虽然叫十大经典排序算法，但却只有九个排序算法。
  2. 视频太长了，特别是冒泡排序的时候，8分钟了才进入正题。
  3. VSCode配置看不懂。
  4. 希尔排序后面代码错了，但是却在后面的排序算法讲解希尔排序的Bug。
  5. 能不能发文档，或链接找不到。
  6. 排序算法学这么多，后来只记得冒泡。

我统一回答一下：

  1. 当时我在录的时候已经准备进入下一个阶段了，所以不想耽搁太久，我以为已经录了桶排序，但实际我没有录，所以就会导致十大排序算法只有九个，但是其实桶排序就是一个思想，像前面计数排序、堆排序、基数排序都算是桶排序。大家也不用纠结这个是十还是九了，因为很大可能会跳入第六个问题，所以不用太在意，知道那意思就行。
  2. 我录视频一般都是一遍过，遇到Bug直接带着改，这样我这边犯错了之后，你那边就不会再犯了，如果想一路顺利走下去的话，可以告诉我，我可以对应出一个“极速版”。然后冒泡排序那个，确实时间久了点，前8分钟都在配VSCode的环境，这部分可以单独拿出来。这次用Java重做我就把它们分得明明白白。
  3. VSCode配置看不懂，其实可以看我之前的视频，我基本很详细很慢地讲，看完你应该就知道怎么配了，如果觉得麻烦其实可以直接用IDE，这样就不用自己配了，这次使用Java重做，我就打算使用IDEA，这样就不用担心什么IDEA不会配这种问题了，当然为了方便我会加一些插件。
  4. 首先希尔排序我肯定是没有讲错的，算法的步骤就是和我视频说的那样，它就是插入排序的升级版，但是复制粘贴疏忽了，不知道那些被它的代码困扰的同学有没有看它后面的快速排序，在快速排序中就修复了希尔排序留下的Bug。然后这次重做，我就把它们分好，这样即使有了错误，我会追加一个视频在后面，也不会把它放到下一个排序算法中讲解。
  5. 一般我是不发markdown文档的，因为我已经发了博客，你完全可以通过我的博客来看文档。如果你实在想要，你可以发邮件给我，我的邮箱cukorzhong@gmail.com，B站的私信功能比较差，我不太喜欢用。
  6. 出现这种情况的根本原因是你为了快速完成排序操作直接调用了高级语言自带的排序算法，然后最初始、最基础的排序算法就被你忘记了，所以就什么都不记得，好一点的就还记得冒泡，这是正常现象。视频是去年录的，我记得也不多了其实，但是复习一下还是可以想得起来的，所以这次要重做咯。一般自带的排序算法底层用的就是快速排序或归并排序，使用自带的可以提高了我们的开发效率，但是原理什么的，我们不能忘记，所以这就是为什么还要学这些最基础的排序算法，而且当你要排序的序列较少的时候冒泡排序要比快速排序这些要快。

进入正题

我们这次重做，使用的环境如下：

  • 编程语言：Java，其他语言自行翻译，都差不多，重要的是排序的思想和步骤。
  • IDE：IDEA2021.3.3
  • 包管理工具（可选）：Maven，其实就是为了方便引入测试的依赖，Junit，也可以自己直接把jar包引进来。
  • 编辑器：SubLime Text、VSCode、nvim、notepad、记事本（Windows自带）等，选其中之一即可。
  • 版本控制（可选）：git
  • 画图工具：draw.io、画图（Windows自带）等，选其中之一即可。
  • 笔记：Typora、uTools插件等，选其中之一即可。
  • 博客：cukor666.github.io（Cukor粉丝自带）
