用途

正则表达式用于字符串匹配，一般用作验证邮箱，验证用户名，验证电话，验证身份证等，关于字符串匹配验证的。

Go标准库提供了正则的包。通过下面的指令可以在控制台终端看到正则表达式的文档。

go doc regexp/syntax

查看官方文档。

第一个正则表达式代码

​x
package main


import (
    "fmt"
    "regexp"
)


func main() {
    // 在初始化阶段，编译表达式一次
    // 使用原始字符串以避免引用反斜杠。反斜杠会转义
    var validID = regexp.MustCompile(`^[a-z]+\[[0-9]+\]$`)


    fmt.Println(validID.MatchString("adam[23]"))
    fmt.Println(validID.MatchString("eve[7]"))
    fmt.Println(validID.MatchString("Job[48]"))
    fmt.Println(validID.MatchString("snakey"))
}
xxxxxxxxxx
true
true
false
false
  • ^，$：开头和结尾
  • [a-z]：任意一个小写字符
  • [a-z]+：任意一个小写字符，出现1次或多次
  • [0-9]+：任意一个数字，出现1次或多次
  • \[，\]：左中括号和右中括号

整个合起来的意思就是，匹配小写字符串开头，后面跟一个中括号，中括号里面是数字的字符串，满足这个规则返回true，否则返回false.

编译

  • func Compile(expr string) (*Regexp, error)
  • func CompilePOSIX(expr string) (*Regexp, error)
  • func MustCompile(str string) *Regexp
  • func MustCompilePOSIX(str string) *Regexp

带有POSIX的表示不使用Perl的规则。带有Must的表示如果出错了会报异常。一般就使用第3个吧，但也看情况。

查找

官方上，那些Find开头的表示查找的函数。简单使用几个。

查找最左边

xxxxxxxxxx
func (re *Regexp) Find(b []byte) []byte

官方：Find returns a slice holding the text of the leftmost match in b of the regular expression. A return value of nil indicates no match.

机翻：返回一个切片，其中包含正则表达式b中最左边匹配的文本。返回值nil表示不匹配。

人话：b是待匹配的正则表达式，这个函数的功能是从左到右开始匹配，只要匹配到了就返回对应的文本，一直匹配不到就返回nil

xxxxxxxxxx
package main


import (
    "fmt"
    "regexp"
)


func main() {
    re := regexp.MustCompile(`foo.?`)
    fmt.Printf("%q\n", re.Find([]byte(`seafood fool`)))
    fmt.Printf("%q\n", re.Find([]byte(`seafoad fool`)))
}
xxxxxxxxxx
"food"
"fool"
  • .：任意一个字符
  • ?：表示它前面的字符出现一次或零次，?前面一定要有东西，不能单独存在。

连起来foo.?就表示以foo开头的字符串接任意字符出现一次或零次。而Find函数只获取最左边的。

查找全部

xxxxxxxxxx
func (re *Regexp) FindAll(b []byte, n int) [][]byte

官方：FindAll is the 'All' version of Find; it returns a slice of all successive matches of the expression, as defined by the 'All' description in the package comment. A return value of nil indicates no match.

机翻：FindAll是Find的“All”版本；它返回表达式的所有连续匹配的片段，如包注释中的“all”描述所定义。返回值nil表示不匹配。

人话：从左到右匹配，只要满足的都存到返回值切片中。n表示字符串的长度，如果传的是-1，它会自动获取。

FindAll源码：

xxxxxxxxxx
func (re *Regexp) FindAll(b []byte, n int) [][]byte {
    if n < 0 {
        n = len(b) + 1
    }
    var result [][]byte
    re.allMatches("", b, n, func(match []int) {
        if result == nil {
            result = make([][]byte, 0, startSize)
        }
        result = append(result, b[match[0]:match[1]:match[1]])
    })
    return result
}

示例：

xxxxxxxxxx
re := regexp.MustCompile(`foo.?`)
fmt.Printf("%q\n", re.FindAll([]byte(`seafood fool`), -1))
xxxxxxxxxx
["food" "fool"]

查找全部匹配下标

xxxxxxxxxx
func (re *Regexp) FindAllIndex(b []byte, n int) [][]int

官方：FindAllIndex is the 'All' version of FindIndex; it returns a slice of all successive matches of the expression, as defined by the 'All' description in the package comment. A return value of nil indicates no match.

机翻：FindAllIndex是FindIndex的“All”版本；它返回表达式的所有连续匹配的片段，如包注释中的“all”描述所定义。返回值nil表示不匹配。

人话：FindIndex的All版本，FindAll的Index版本。获取全部匹配的子串对应的下标。

xxxxxxxxxx
content := []byte("Londno")
re := regexp.MustCompile(`o.`)
fmt.Println(re.FindAllIndex(content, 1))
fmt.Println(re.FindAllIndex(content, -1))
xxxxxxxxxx
[[1 3]]
[[1 3]]

官方文档给的结果是错的，本地运行的结果如上所示。

因为它传-1进去会变成n = len(b) + 1，这个-1并不是从尾部开始匹配的意思。

结果中，1表示的是匹配子串的开始下标，3表示子串的长度（从左边开始算）。
